### 1.3 测试框架

| 框架 | 用途 | 适用范围 |
|------|------|---------|
| XCTest | 单元测试 + 性能测试 | 模块逻辑（粘贴流程协调器、辅助功能服务、剪贴板写入模块、剪贴板消费监听器、应用设置）、mock 测试 |
| XCUITest | UI 自动化测试 | 全局快捷键呼出、面板交互（单击/双击/方向键/回车/Esc）、失焦关闭、降级浮层显示、设置面板配置 |
| 手动 | 人工验证 | 有权限路径真实粘贴（模拟按键到外部应用）、caret 定位、权限撤销降级、截图/录屏 |

## 5. 测试执行计划

### 5.1 按 Phase 拆分测试执行

| Phase | 实现内容 | 对应用例 | 测试框架 |
|-------|---------|---------|---------|

3. **Phase 3 实现完成后**：执行 Phase 3 用例（降级路径）+ 回归 Phase 1/2 用例
4. **Phase 4 实现完成后**：执行 Phase 4 用例（有权限路径）+ 回归 Phase 1/2/3 用例 + 全量 NFR 验证
