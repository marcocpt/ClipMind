## 1.2 复用的 10 项需求摘要（来自上游文件）

1. **用户问题和业务目标**：用户在任意应用准备粘贴时操作路径过长；目标为按全局快捷键在 caret/鼠标附近呼出快速粘贴面板，单击选中、双击粘贴，无需切换应用上下文。
2. **成功标准**：cmd+shift+v 在 caret/鼠标附近出现 Popover；单击高亮、双击粘贴；有权限自动粘贴、无权限写剪贴板+浮层提示；浮层消费后或 5s 超时消失；AC 通过 XCTest/XCUITest + 手动验收。
3. **范围边界**：必须改 GlobalHotkeyService、新建独立 QuickPasteWindow、运行时检测权限、ClipRowView 增加交互、键盘导航、失焦/粘贴后自动关闭；明确不改主窗口、不支持图片/文件路径自动粘贴、不支持多选、不改快捷键配置 UI、不实现二次快捷键 toggle。
4. **用户流程**：入口 cmd+shift+v → 检测权限 → 定位 Popover → 浏览列表 → 单击选中 → 双击粘贴（有/无权限分支）→ 失败路径（权限撤销/caret 缺失/剪贴板写入失败）→ 退出条件（Esc/失焦/粘贴完成）。
5. **数据和接口**：新增 QuickPastePanelController/AccessibilityService/PasteSimulator/PasteOverlayController + 通知 `.openQuickPaste`；修改 GlobalHotkeyService.handleHotkeyPressed/ClipRowView/PopoverView→QuickPasteView；AppSettings 新增 `quickPasteOverlayDuration = 5.0`。
6. **兼容和迁移**：cmd+shift+v 默认行为从唤起主窗口改为呼出 Popover A；主窗口入口改为菜单栏点击或"查看全部"按钮；保留快捷键配置能力；保留菜单栏 StatusItem toggle 行为；老用户无感知迁移。
7. **AC（12 条）**：AC-F1.9-1 ~ AC-F1.9-12，覆盖快捷键呼出、caret 定位、屏幕中央定位、单击选中、双击粘贴、有权限粘贴流程、无权限降级流程、Esc 关闭、失焦关闭、粘贴后关闭、图片/文件路径提示、权限撤销降级。
8. **阶段拆分（4 Phase）**：Phase 1（独立 Popover + 屏幕中央定位，合规基线，低风险）/ Phase 2（ClipRowView 单击/双击 + 键盘导航，中风险）/ Phase 3（无权限降级粘贴流程，中风险）/ Phase 4（Accessibility 权限路径，高合规风险，独立便于回退）。
9. **UI 证据**：XCUITest（快捷键呼出、单击/双击、键盘导航、Esc/失焦关闭、浮层提示出现与消失）+ 手动验收（备忘录/Safari 真实应用验证）+ 截图/录屏（保存到 `screenshots/` 和 `recordings/`）。
10. **文档位置与编号**：F1.9 / P0 / `docs/planning/P0/F1/` / 编写顺序需求→设计→原型→测试用例 / history 日志 `2026-07-23-F1.9-快捷粘贴面板.md`。
