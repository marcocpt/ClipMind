## 必须修复的问题

### 问题 #1：MVP 路径下 T4.1/T4.4/T4.5 对 T3.7 的依赖冲突

**涉及任务**：T4.1（Web 交互预览页）、T4.4（截图 + Session ID 收集）、T4.5（最终 .app 构建）

**具体问题**：
- T4.1 依赖标注为 `T2.5, T3.7`
- T4.4 依赖标注为 `T2.5, T3.7`
- T4.5 依赖标注为 `T3.7, T4.1`
- 但 T3.7 在 MVP 兜底路径中被标记为"⚡ 可简化（MVP 跳过）"
- T4.1/T4.4/T4.5 均标记为"✅ 必做"

这导致 MVP 路径下 3 个必做任务的依赖无法满足，开发者无法确定是否可以执行这些任务。

**为什么这对执行很重要**：
当开发者切换到 MVP 路径时，会面临矛盾：T3.7 被跳过，但 T4.1/T4.4/T4.5 又依赖 T3.7 且必须完成。如果不明确处理方式，开发者可能：
1. 错误地认为 T4.1/T4.4/T4.5 也需要跳过
2. 或者尝试执行时发现依赖缺失而卡住

**建议修复方式**（任选其一）：
- **方案 A**：将 T4.1/T4.4/T4.5 对 T3.7 的依赖改为"可选依赖"（MVP 路径下不依赖）。具体说明：
  - T4.1 Web 预览页：Web 页是独立 HTML/JS，不依赖 .app 的首启引导代码，依赖 T3.7 可改为无依赖或仅依赖 T2.5
  - T4.4 截图：MVP 路径下截图不包含首启引导界面，依赖 T3.7 可改为可选
  - T4.5 .app 构建：MVP 路径下 .app 不包含首启引导代码，依赖 T3.7 可改为可选
- **方案 B**：在 MVP 兜底路径章节（第 6 章）明确说明："MVP 路径下，T4.1/T4.4/T4.5 的依赖 T3.7 自动解除，Web 预览页和 .app 不包含首启引导流程"

### 问题 #2：T1.3 输出 TypeEmbeddings.swift 内容描述缺失

**涉及任务**：T1.3（ClassificationService.swift）

**具体问题**：
- T1.3 输出列出了 `ClipMind/Classify/TypeEmbeddings.swift`
- 但任务描述（"11 种入库类型向量匹配 + 阈值判断"）未说明 TypeEmbeddings.swift 的内容
- 开发者无法确定：11 种类型的向量是预计算常量？运行时通过 embed() 生成？存储在文件中？硬编码在 Swift 代码中？
- 设计规范 5.2.3 节仅提到"与 11 种入库类型向量计算余弦相似度"，未说明类型向量的来源

**为什么这对执行很重要**：
TypeEmbeddings.swift 是分类服务的核心依赖——ClassificationService 需要用 11 种类型的向量与内容向量做余弦相似度比较。如果开发者不知道这些向量从哪里来，就无法实现分类逻辑。可能的选择包括：
1. 运行时用 embed() 对 11 种类型的描述文本生成向量（如 embed("代码片段"), embed("URL 链接") 等）
2. 预计算并硬编码为 Swift 常量数组
3. 预计算存入 JSON 文件运行时加载

不同选择会影响实现方式和测试方式。

**建议修复方式**：在 T1.3 任务描述中补充 TypeEmbeddings.swift 的实现说明，例如：
> "TypeEmbeddings.swift：11 种入库类型的预计算向量常量。在初始化时通过 LocalEmbeddingService.embed() 对每种类型的代表文本（如 code 类型用 '代码片段 函数定义 语法关键字'）生成向量并缓存，或预计算为静态常量数组。"

## 建议（仅供参考，不阻止通过）

### 建议 1：T1.1 工时建议调整为 4h

T1.1（嵌入模型准备）涉及 HuggingFace 模型下载、ONNX→CoreML 转换脚本编写、可能的兼容性调试。设计规范 12.4 节将"嵌入模型最终选择"列为待确认问题，12.1 节将"Core ML 模型转换失败"列为技术风险（概率低但影响大）。3h 可能不足以应对转换过程中的调试。建议调整为 4h，与风险章节的"coremltools 转换脚本预留 3h 工时"保持一致并增加 1h 缓冲。

### 建议 2：T1.5/T1.7 的"双人交叉抽查"不适用于单人项目

T1.5（分类测试集）和 T1.7（搜索测试集）的验证方式包含"双人交叉抽查 10% 样本"。初赛为个人项目，该步骤无法执行。建议改为"自审 + 间隔 1 天后复审"或"对边界样本单独标注复核"。

### 建议 3：T0.5 AppDetector 依赖可简化
