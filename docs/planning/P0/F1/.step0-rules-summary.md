## 0.4 关键约束（来自初赛要求 + 报名帖）

### 初赛交付要求（硬约束）
1. **可交互 Demo**：核心功能能跑通、能让人体验到价值
2. **基于 TRAE IDE 或 TRAE Work 完成**：开发过程必须用 TRAE
3. **可体验地址或 HTML 文件**：部署随时可公开访问的链接，或交互式 HTML（Zip 打包上传），硬件赛道可视频
4. **Demo 作品帖**：包含 4 部分（Demo 简介 / 创作思路 / 体验地址 / TRAE 实践过程）
5. **截图 ≥ 3 张**：开发关键步骤截图
6. **Session ID ≥ 3 个**：证明作品由 TRAE 开发完成
7. **赛道标签**：生活娱乐/学习工作/社会服务/硬件交互 四选一

### 评审 4 维度（隐含约束）
- 评审按 4 个维度打分（具体维度见赛事细则，本文档关注"核心功能可体验"）

### ClipMind 产品约束（来自报名帖与 HTML）
1. **平台**：macOS 原生桌面 App
2. **形态**：独立应用（非插件/非菜单栏小工具单一形态）
3. **AI 能力**：自动分类、语义搜索、一键处理均依赖 AI 模型
4. **隐私架构**：本地优先（local-first），AES-256 加密，敏感识别
5. **初赛范围**：Phase 01 MVP，不含模板/同步/工作流引擎
