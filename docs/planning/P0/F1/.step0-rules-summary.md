### 设计规范关键信息提取

#### Phase 拆分（来自设计规范第 11 节）
| Phase | 目标 | 范围 | AC 覆盖 |
|-------|------|------|---------|
| Phase 0 | 基础设施 | 剪贴板监听 + 本地加密存储 + 菜单栏 UI 骨架 | AC-01, AC-04, AC-18, AC-23(部分) |
| Phase 1 | 核心 AI | 11 种入库类型分类 + 敏感识别 + 语义搜索 | AC-05~AC-12 |
| Phase 2 | 一键处理 | 4 种处理 + API Key 多提供商配置 | AC-13~AC-17 |
| Phase 3 | 隐私与设置 | 敏感识别 + 黑名单 + 自动清理 + 设置面板 + 首启引导 | AC-08, AC-20~AC-24 |
| Phase 4 | Web + Demo 帖 | Web 预览页 + Demo 作品帖 + 截图 + Session ID | AC-25 |

#### Phase 依赖（来自设计规范 11.6）
- Phase 0 是所有后续阶段的基础（必须先完成）
- Phase 1 和 Phase 3 可并行（Phase 1 依赖 Phase 0，Phase 3 仅依赖 Phase 0 的 UI 骨架）
- Phase 2 依赖 Phase 1（处理需要分类后的内容）
- Phase 4 依赖 Phase 2 和 Phase 3

#### 测试用例统计（来自测试用例表 v1.1）
- AC 总数：25
- 测试用例总数：69
- 按框架：XCTest 41 / XCUITest 11 / 手动 16 / curl 1
- 所有用例当前状态：❌ MISSING

#### 关键技术决策（来自设计规范 1.5）
- Demo 交付形态：Swift 原生 .app + Web 交互预览页
- AI 能力：本地嵌入模型（all-MiniLM-L6-v2）+ 云端 LLM API 混合
- UI 形态：菜单栏常驻（NSStatusItem）+ 主窗口（SwiftUI）
- API Key：多提供商（OpenAI/智谱/通义/DeepSeek）
- 隐私默认：敏感识别 + 应用黑名单 + 自动清理全部开启
- 测试策略：XCTest + XCUITest + SwiftLint，核心模块 70%+ 覆盖

#### 数据模型（来自设计规范 5.1）
- ClipItem（剪贴板条目）：id, content, contentType, sourceApp, timestamp, summary, translation, rewrite, todos, embeddings
- AppSettings（应用设置）：apiProvider, apiKey, sensitiveDetectionEnabled, appBlacklist, autoCleanupEnabled, cleanupDays, launchAtLogin, hotkey
- BlacklistEntry（黑名单条目）：id, bundleId, appName, addedAt, isDefault

#### 持久化方案（来自设计规范 5.4）
- 加密 SQLite（CryptoKit + SQLite.swift 手动加密）
- 数据库路径：`~/Library/Application Support/ClipMind/clipmind.db`
- 加密算法：AES-256-GCM
- API Key 存储到 Keychain

## 0.4 关键约束

### 初赛硬约束（来自初赛要求.md）
1. **截止日期**：2026-07-15
2. **可交互 Demo**：核心功能能跑通、能让人体验到价值

### 实施计划约束
- **任务粒度**：每个任务应可在 1 个 TRAE Session 内完成
- **AC 映射**：每个任务必须映射到至少 1 条 AC
- **测试映射**：每个任务必须映射到至少 1 个测试用例
- **依赖明确**：任务间依赖关系必须清晰可拓扑排序
- **验证方式**：每个任务必须有明确的完成验证方式（测试命令或手动验证步骤）
