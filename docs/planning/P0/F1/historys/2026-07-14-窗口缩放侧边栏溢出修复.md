# 窗口缩放侧边栏溢出修复 history

> 日期：2026-07-14 | 功能：F1.14 窗口外层 frame 嵌套冲突修复

## 变更摘要

修复"窗口缩小宽度时，搜索栏和下面的列表会移出显示窗口，不应该移动左边导航栏"bug。F1.12 修复了 `MainWindow` 内层 frame（minWidth 700→980），但未同步调整 `ClipMindApp` 外层 frame（minWidth 仍为 900），导致外层 frame < 内层 frame 的嵌套冲突。窗口实际最小宽度 900 小于内层期望 980，缩窗时 `NavigationView` 侧边栏内容被推出窗口可见区域。

## 根因

`ClipMindApp.swift` 第 14 行（修复前）：

```swift
MainWindow()
    .frame(minWidth: 900, minHeight: 600)
```

`MainWindow.swift` 内层使用 `LayoutConstants.mainWindowMinWidth = 980`。SwiftUI 嵌套 frame 中，外层 frame minWidth(900) 小于内层 frame minWidth(980) 时，NSSplitView 实际可分配宽度被外层约束压到 900，内层 NavigationView 侧边栏（minWidth 350）+ 详情面板总宽度需求 980 被压缩，侧边栏内容溢出窗口可见区域。

## 修复方案

1. 在 `LayoutConstants` 中新增命名常量 `appWindowMinWidth: CGFloat = 980` 与 `appWindowMinHeight: CGFloat = 500`，与内层对齐消除嵌套冲突
2. `ClipMindApp.swift` 外层 frame 字面量替换为命名常量引用，消除魔术数字
3. 新增两个 XCTest 不变量测试防止回归

## 代码变更

- `ClipMind/UI/MainWindow/LayoutConstants.swift`：新增 `appWindowMinWidth = 980`、`appWindowMinHeight = 500` 两个命名常量（含文档注释说明 F1.14 由原值 900/600 调整）
- `ClipMind/App/ClipMindApp.swift`：外层 frame 从字面量 `(minWidth: 900, minHeight: 600)` 替换为 `(minWidth: LayoutConstants.appWindowMinWidth, minHeight: LayoutConstants.appWindowMinHeight)`
- `ClipMindTests/UI/MainWindowLayoutTests.swift`：新增 `testAppWindowMinWidthNotSmallerThanMainWindowMinWidth` 与 `testAppWindowMinHeightNotSmallerThanMainWindowMinHeight` 两个不变量测试，断言外层 ≥ 内层
