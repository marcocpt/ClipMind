# F1.9 Phase 4 Accessibility 路径完成

> 最后更新：2026-07-23 | 版本：v1.0

## 变更摘要

F1.9 快捷粘贴面板 Phase 4（Accessibility 路径）TDD 实现完成。实现有辅助功能权限时的自动粘贴路径：检测权限 → 写剪贴板 → 关闭面板 → 模拟系统标准 Cmd+V 粘贴按键到前台应用。同时实现 caret 附近面板定位（有 caret 用 caret，无 caret 降级鼠标位置，无权限降级屏幕中央/上次位置）。所有有权限路径代码通过 `#if CLIPMIND_DEV` 编译条件隔离，主 Scheme `ClipMind` 保持 App Store 完全合规（行为与 Phase 3 一致，所有粘贴流程走降级浮层路径）。

## 合规说明

- **主 Scheme `ClipMind`**：不编译 `CLIPMIND_DEV` 条件代码，有权限路径回退到显示降级浮层（Fix 10 合规回退，Phase 3 已实现），App Store 完全合规
- **Dev Scheme `ClipMind-Dev`**：编译 `CLIPMIND_DEV` 条件代码，有权限路径调用 `PasteSimulator.simulatePaste()` 模拟 Cmd+V，仅用于本地验证与技术可行性评估
- **`ClipMind-Dev` 不归档发布构建**（AGENTS.md 第 10 节），archive 步骤跳过
- 所有有权限路径相关代码（AccessibilityService、PasteSimulator、CaretPanelLocator、PasteCoordinator 扩展、AppDelegate 切换）均用 `#if CLIPMIND_DEV` 包裹

## 关键设计决策

- **`#if CLIPMIND_DEV` 编译条件隔离**：所有有权限路径代码用编译条件包裹，主 Scheme 编译时这些代码被排除，确保 App Store 合规
- **`DebugDev` build configuration**：基于 `debug` 类型，额外定义 `SWIFT_ACTIVE_COMPILATION_CONDITIONS: CLIPMIND_DEV`，`ClipMind-Dev` Scheme 使用此配置
- **`pasteSimulator: Any?` 类型**：避免在 `init` 参数上使用 `#if CLIPMIND_DEV` 条件编译，主 Scheme 传 nil，Dev Scheme 传 `PasteSimulator` 实例
- **CaretPanelLocator 降级链**：caret 位置 → 鼠标位置 → 上次关闭位置 → 屏幕中央，四级降级确保面板总能显示
- **`screenFrameProvider` closure 注入**：CaretPanelLocator 测试通过注入 closure 模拟不同屏幕尺寸，避免依赖真实 NSScreen
- **PermissionRequester.axTrustedCheck 改为 closure**：从 static 方法改为 static closure 属性，测试可临时替换实现，测试后恢复默认
- **PasteSimulator UI 测试模式**：检测 `--UITEST_QUICK_PASTE_PANEL` 启动参数，跳过 CGEvent 发送（面板关闭后前台应用不可预测，真实 Cmd+V 会激活其他应用并干扰 XCUITest），仅记录 UserDefaults 标记供 UI 测试验证
