## 关键决策记录

### 决策 1：注入失败清理逻辑

- **背景**：SampleDataSeeder.seedIfNeeded 无事务包裹，部分失败后 countSamples() > 0 导致下次跳过
- **决策**：catch 块中调用 `try? store.deleteSamples()` 清理已写入数据
- **理由**：确保注入失败可重试，符合 4.1 状态机设计

### 决策 2：搜索关键词 fallback

- **背景**：NLEmbedding 英文模型对中文匹配度不足
- **决策**：单元测试改用英文关键词验证语义搜索；UI 测试保留中文"报错"验证文本匹配
- **理由**：单元测试验证 SearchService 语义搜索能力（英文关键词可靠）；UI 测试验证 MainWindow.performSearch 文本匹配（中文可行）
- **后续**：待接入多语言 embedding 模型后补充中文语义搜索验证

### 决策 3：UI 测试数据库清理

- **背景**：FirstLaunchUITests 完成引导后触发示例数据注入，残留到后续 PopoverUITests/MainWindowUITests
- **决策**：所有 UI 测试类添加 `cleanUpDatabase()` 和 `tearDown()`
- **理由**：确保每个测试从空数据库开始，避免状态污染
