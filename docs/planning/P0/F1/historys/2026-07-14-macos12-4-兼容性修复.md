# macOS 12.4 兼容性修复 history

> 日期：2026-07-14 | 功能：F1.13 macOS 12.4 部署目标兼容

## 变更摘要

将 `MACOSX_DEPLOYMENT_TARGET` 从 13.0 降为 12.4 后，`GeneralSettingsView.updateLaunchAtLogin(_:)` 调用的 `SMAppService`（macOS 13+）导致 6 处编译错误。修复方案是用 `if #available(macOS 13.0, *)` 包裹 `SMAppService` 调用，macOS 12.x 降级为仅记录日志。

## 根因

`SMAppService` 属于 ServiceManagement 框架，仅 macOS 13+ 可用。原代码在 `updateLaunchAtLogin` 中无条件调用 `SMAppService.mainApp.register()` / `unregister()`，当部署目标降为 12.4 时编译器拒绝该 API 调用。

## 修复方案

在 `GeneralSettingsView.updateLaunchAtLogin(_:)` 中用 `if #available(macOS 13.0, *)` 分支：
- macOS 13+：调用 `SMAppService.mainApp.register/unregister`，行为不变
- macOS 12 及以下：记录日志，跳过实际注册（MVP 阶段暂不支持 `SMLoginItemSetEnabled` helper bundle 方案）

UI 测试早返回逻辑（`--UITEST_SHOW_MAIN_WINDOW`）保持不变。

## 代码变更

- `project.yml`：`deploymentTarget.macOS` 13.0 → 12.4；`MACOSX_DEPLOYMENT_TARGET` 13.0 → 12.4
- `ClipMind.xcodeproj/project.pbxproj`：2 处 `MACOSX_DEPLOYMENT_TARGET = 13.0` → `12.4`
- `ClipMind/UI/Settings/GeneralSettingsView.swift`：`updateLaunchAtLogin(_:)` 增加 `if #available(macOS 13.0, *)` 分支与 macOS 12 fallback 日志
