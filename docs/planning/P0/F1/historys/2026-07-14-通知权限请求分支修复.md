# 通知权限请求分支修复 history

> 日期：2026-07-14 | 功能：F1.11 通知权限设置

## 变更摘要

修复"点击授权通知按钮没有弹出通知设置"bug。用户曾拒绝通知权限后，macOS 不再弹出授权对话框，导致点击按钮无反应。

## 根因

`PermissionRequestView.requestNotificationPermission()` 直接调用 `UNUserNotificationCenter.requestAuthorization`，不检查 `authorizationStatus`。当 status 为 `.denied` 时，系统直接返回 granted=false 不弹窗，用户看到点击按钮无反应。

## 修复方案

在 `PermissionRequester` 中新增 `requestNotification(completion:)` 方法，根据 `authorizationStatus` 分支处理：
- `.notDetermined`：调用 `requestAuthorization` 弹出系统对话框
- `.denied`：打开系统设置通知页面引导用户手动开启
- `.authorized`/`.provisional`/`.ephemeral`：不操作

## 代码变更

- `ClipMind/Privacy/PermissionRequester.swift`：新增 3 个可注入闭包 + `requestNotification(completion:)` 方法
- `ClipMind/UI/Onboarding/PermissionRequestView.swift`：`requestNotificationPermission()` 委托给 `PermissionRequester.requestNotification`
- `ClipMindTests/Privacy/PermissionRequesterTests.swift`：新增 3 个 XCTest 覆盖三种授权状态
