# 步骤 1 需求摘要（F1 · ClipMind 初赛 MVP）

> 创建时间：2026-07-12 | 通过 grill 确认 | 对应特性：F1.x（Phase 01 · P0）

## 1. 用户问题和业务目标

### 用户问题
macOS 用户每天复制大量信息（代码、链接、报错、文案、会议内容、资料），普通剪贴板只是临时容器：
- 信息被新复制内容覆盖，重要内容难以找回
- 无法理解内容含义，只按时间排列
- 复制长文本后需手动总结、翻译、改写、整理
- 剪贴板可能包含密码/Token/验证码等敏感信息，缺少智能识别与保护

### 业务目标
- 初赛目标：交付可交互、能体验的 Demo，让评审一眼看懂 ClipMind 的价值
- 把 macOS 剪贴板从"临时复制工具"升级为"AI 个人知识管家"
- 让复制过的信息可找回、可理解、可复用

## 2. 成功标准（初赛评审维度）

- 核心功能能跑通、能让人体验到价值
- 基于 TRAE IDE 完成，附 Session ID ≥ 3 个
- 提供可体验地址（.app 下载 + Web 交互预览）
- Demo 作品帖结构完整（简介/创作思路/体验地址/TRAE 实践过程）
- 关键步骤截图 ≥ 3 张
- 赛道标签：学习工作（待最终确认）

## 3. 范围边界

### 必须做（F1.x · Phase 01 · P0）
- F1.1 剪贴板监听与捕获
- F1.2 自动分类（12 种类型）
- F1.3 自然语言语义搜索
- F1.4 一键处理（总结/翻译/改写/提取待办）
- F1.5 本地加密存储（AES-256）
- F1.6 隐私保护（敏感识别/应用黑名单/自动清理）

- iOS/iPadOS 版本

## 4. 用户流程和交互入口

### 交互入口
- 菜单栏常驻图标（NSStatusItem）
- 点击菜单栏图标 → popover 显示最近几条剪贴内容
- "查看全部" → 打开主窗口（完整历史/搜索/设置）

### 主要路径
复制任意内容 → ClipMind 自动捕获 → AI 自动分类 → 入库加密存储 →
用户通过 popover 或主窗口查看 → 自然语言搜索 / 一键处理 → 复用

### 首次启动路径
启动 .app → 权限请求（辅助功能/通知）→ API Key 配置引导（可选）→
隐私默认值提示（敏感识别/黑名单/30天清理已开启）→ 进入主界面

### 失败路径
- 未配置 API Key：分类/搜索本地可用，总结/翻译/改写/待办置灰并提示"需配置 API Key"
- 剪贴板内容被识别为敏感：不入库，仅提示"已忽略敏感内容"
- 来源 App 在黑名单：自动忽略，不提示
- 云端 API 调用失败：提示错误，不阻塞本地功能

## 5. 数据和接口

### 数据模型（剪贴板条目）
- id: UUID
- content: String（文本）/ Data（图片）/ URL（文件路径）
- contentType: ContentType（12 种枚举）
- sourceApp: String（来源 App bundleId）
- timestamp: Date
- summary: String?（AI 总结，调用后生成）
- translation: String?（翻译结果）
- todos: [TodoItem]?（提取的待办，含任务/负责人/截止时间）
- isSensitive: Bool（已忽略的敏感条目标记）
- tags: [String]
- embeddings: [Float]?（本地嵌入向量，用于语义搜索）

### 接口
- 系统接口：NSPasteboard（监听剪贴板）、NSWorkspace（获取前台 App）
- AI 接口：本地嵌入模型（Core ML/MLX）、云端 LLM API（OpenAI/智谱/通义/DeepSeek）
- 持久化：本地加密数据库（AES-256）
- 配置：UserDefaults / 加密配置文件（API Key）

## 6. 兼容性和迁移

### 兼容性
- 新项目，无历史数据迁移
- macOS 版本要求：macOS 14+（Sonoma），需确认 SwiftUI/SwiftData 可用性
- 架构：Universal Binary（Intel + Apple Silicon）

### 迁移策略
- 不适用（新项目）

## 7. 验收标准 AC（概要，详见设计规范）

- AC-01：复制文本后 3 秒内出现在 popover 与主窗口历史
- AC-02：12 种类型自动分类准确率 ≥ 80%（抽样测试）
- AC-03：自然语言搜索返回结果 < 500ms，Top-5 命中率 ≥ 70%
- AC-04：一键总结/翻译/改写/提取待办在配置 API Key 后可用
- AC-05：本地存储使用 AES-256 加密，数据文件无法直接读取
- AC-06：密码/Token/验证码自动识别并忽略
- AC-07：1Password/银行 App 复制内容自动忽略
- AC-08：30 天前内容自动清理
- AC-09：菜单栏图标常驻，点击弹出 popover
- AC-10：Web 交互预览页可访问，模拟核心流程

## 8. 阶段拆分（5 阶段递进）

| Phase | 目标 | 范围 | 交付物 |
|-------|------|------|--------|
| Phase 0 | 基础设施 | 剪贴板监听 + 本地加密存储 + 菜单栏 UI 骨架 | 可运行 .app，能捕获与存储 |
| Phase 1 | 核心 AI | 12 类型自动分类 + 语义搜索 | 分类与搜索可用 |
| Phase 2 | 一键处理 | 总结/翻译/改写/提取待办 + API Key 配置 | 处理功能可用 |
| Phase 3 | 隐私与设置 | 敏感识别/应用黑名单/自动清理/设置面板 | 隐私保护完整 |
| Phase 4 | Web + Demo 帖 | Web 交互预览页 + Demo 作品帖 + 截图 + Session ID | 初赛提交物完整 |
