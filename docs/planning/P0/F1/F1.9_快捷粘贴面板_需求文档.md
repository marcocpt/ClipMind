## 1. Background（背景）

ClipMind 已交付初赛 MVP（F1.1~F1.7）：剪贴板监听与捕获、自动分类、语义搜索、一键处理、本地加密存储、隐私保护、菜单栏常驻 + 主窗口交互。用户可以通过菜单栏图标点击弹出 popover 浏览历史，或通过"查看全部"按钮打开主窗口进行搜索与处理。

但是，**用户在任意应用准备粘贴时，操作路径过长**：用户必须先切换到 ClipMind 菜单栏图标或主窗口，选中条目，再切换回目标应用粘贴。这种"应用切换 + 鼠标移动 + 二次切换"的路径破坏了"复制—粘贴"的肌肉记忆，尤其在频繁粘贴历史内容的场景（开发者复用代码片段、文案工作者复用文本模板、客服复用回复话术）下体验割裂。

F1.9 快捷粘贴面板的目标是把"粘贴历史内容"这一动作从"切换应用上下文"中解放出来：用户在任意应用准备粘贴时，按全局快捷键直接在粘贴目标附近呼出快速粘贴面板，单击选中、双击粘贴，无需切换应用上下文。

本特性沿用 F1 主设计规范的剪贴板捕获、加密存储、菜单栏基础设施，不引入新的数据模型或持久化方案，仅在交互层增加"快捷粘贴面板"入口与粘贴流程。

## 2. Problem Statement（问题定义）

**核心问题**：当前 ClipMind 的粘贴路径需要用户离开当前应用上下文，破坏"复制—粘贴"的肌肉记忆，导致：

1. **操作路径过长**：用户在备忘录/Safari/代码编辑器准备粘贴时，需移动鼠标到菜单栏 ClipMind 图标 → 点击弹出 popover → 选条目 → 切换回原应用 → 手动按系统 Cmd+V 粘贴。至少 4 步操作 + 2 次应用切换。
2. **菜单栏 popover 位置绑定**：现有 popover 强制绑定在菜单栏 StatusItem 按钮位置，离用户实际粘贴的 caret 位置远，鼠标移动距离大。
3. **无自动粘贴能力**：现有 popover 仅"复制到剪贴板"，用户仍需手动按 Cmd+V 完成最后一步粘贴，未真正闭环。
4. **无键盘导航**：现有列表行无任何交互（仅展示），用户无法用方向键浏览、回车选中，全靠鼠标点击。

**影响用户**：所有需要频繁复用历史剪贴内容的用户，尤其是开发者（复用代码片段/报错日志）、文案工作者（复用文本模板）、客服（复用回复话术）。

**不解决会怎样**：ClipMind 的"复用"价值停留在"找回"层面，无法兑现"快速复用"的承诺；用户在真实工作流中仍会绕过 ClipMind 手动重写或重搜，留存与活跃受损。

## 3. Goals（目标）

### 3.1 用户目标

- 用户在任意应用准备粘贴时，按全局快捷键直接在粘贴目标附近呼出快速粘贴面板。
- 列表显示历史剪贴项，单击高亮选中、双击粘贴到前台应用。
- 有辅助功能权限时自动粘贴；无权限时写入剪贴板并显示浮层提示引导用户手动粘贴。
- 浮层提示在剪贴板被消费后消失，或超时兜底消失。
- 所有 AC 通过自动化测试 + 手动验收验证。

### 3.2 业务目标

- 把 ClipMind 的"复用"价值从"找回"升级为"快速复用"，闭环"复制—呼出—粘贴"工作流。
- 不引入新的数据模型或持久化方案，沿用 F1 主设计规范的剪贴板捕获与加密存储基础设施。
- 保持 App Store 合规：主 Scheme 在沙盒内实现无权限降级方案；有权限方案使用公开 API + 标准 TCC 权限流程，合规风险独立标注。

### 3.3 成功标准

- 用户在任意应用中按全局快捷键，在 caret 位置或屏幕中央出现快速粘贴面板。
- 列表行支持单击高亮选中、双击粘贴、键盘方向键移动选中、回车粘贴、Esc 关闭。
- 有辅助功能权限时双击自动粘贴到前台应用；无权限时双击写剪贴板并显示浮层提示。
- 浮层提示在剪贴板被消费后消失，或超时兜底消失。
- 失焦自动关闭、粘贴后自动关闭。
- 12 条 AC 全部通过（自动化测试 + 手动验收）。

## 4. Scope（范围）

### 4.1 必须做

- 改造全局快捷键触发行为：默认快捷键从"唤起主窗口"改为"呼出独立快速粘贴面板"。
- 新建独立快速粘贴面板窗口（不依赖菜单栏 StatusItem 按钮位置），可定位到任意屏幕坐标。
- 运行时检测辅助功能权限：有权限走 caret 定位 + 自动粘贴；无权限走屏幕中央或上次关闭位置 + 浮层提示降级方案。
- 复用现有列表行视图骨架，增加单击选中、双击粘贴、键盘方向键导航、回车粘贴交互。
- 失焦自动关闭、粘贴后自动关闭、Esc 关闭。
- 浮层提示：无权限降级时显示"已复制，按 Cmd+V 粘贴"浮层，监听剪贴板被消费后消失，或超时兜底消失。
- 新增设置项：浮层超时兜底时长（默认 5 秒）。

### 4.2 必须不做（详见第 9 节 Out of Scope）

- 不改动主窗口功能（主窗口入口改为菜单栏点击或"查看全部"按钮）。
- 不支持图片/文件路径类型的自动粘贴（显示但双击提示"仅支持文本粘贴"）。
- 不实现多选批量粘贴。
- 不修改全局快捷键配置 UI（保留现有快捷键录制器）。
- 不实现二次快捷键 toggle 关闭（保持单向触发）。

## 5. Functional Requirements（功能需求）

### FR-001：全局快捷键触发呼出快速粘贴面板

**需求**：用户在任意应用准备粘贴时，按默认全局快捷键直接呼出独立快速粘贴面板窗口（不再是主窗口）。

**行为**：

- 默认快捷键保持现有配置不变（即沿用现有默认值）。
- 触发时发送"打开快速粘贴面板"信号，由应用主代理接收并调起独立快速粘贴面板窗口。
- 不影响菜单栏 StatusItem 的 toggle popover 行为（菜单栏点击仍弹出原 popover）。
- 不影响主窗口入口（主窗口通过菜单栏 popover 的"查看全部"按钮或菜单栏点击进入）。

### FR-002：快速粘贴面板窗口独立定位

**需求**：快速粘贴面板是独立窗口，不依赖菜单栏 StatusItem 按钮位置，可定位到任意屏幕坐标。

**行为**：

- 有辅助功能权限时：获取前台应用当前 caret 位置，面板显示在 caret 附近。若前台应用无 caret（如文件管理器），降级到鼠标当前位置附近。
- 无辅助功能权限时：面板显示在屏幕中央，或上次关闭时记录的位置（若上次位置仍在屏幕可视范围内）。
- 面板尺寸固定（与现有菜单栏 popover 视觉一致），不超出屏幕可视范围。
- 面板出现后立即获得键盘焦点，接收键盘导航。

### FR-003：列表行单击高亮选中

**需求**：快速粘贴面板的列表行支持单击高亮选中。

**行为**：

- 单击某行：该行高亮显示，其他行取消高亮。
- 同一时刻最多一行高亮（单选模式）。
- 高亮状态有明确的视觉差异（背景色/边框）。
- 高亮行可通过键盘方向键上下移动。

### FR-004：键盘方向键导航

**需求**：快速粘贴面板支持键盘方向键上下移动高亮选中行。

**行为**：

- 上方向键：高亮行上移一行（若已在第一行则不动）。
- 下方向键：高亮行下移一行（若已在最后一行则不动）。
- 面板出现时默认高亮第一行（若列表非空）。
- 方向键导航不影响列表滚动（必要时自动滚动以保持高亮行可见）。

### FR-005：列表行双击触发粘贴流程

**需求**：列表行双击触发粘贴流程（有权限自动粘贴 / 无权限降级）。

**行为**：

- 双击文本类型行：进入粘贴流程（FR-006 或 FR-007）。
- 双击图片/文件路径类型行：显示提示"仅支持文本粘贴"，不进入粘贴流程。
- 双击后立即关闭快速粘贴面板。

### FR-006：有辅助功能权限时自动粘贴

**需求**：当应用已获得辅助功能权限时，双击文本类型行自动粘贴到前台应用。

**行为**：

- 检测辅助功能权限状态（运行时检测，不弹 TCC 提示对话框）。
- 有权限时：写入剪贴板 → 关闭快速粘贴面板 → 模拟系统粘贴按键到前台应用。
- 模拟粘贴按键后，前台应用 caret 位置插入剪贴板文本内容。
- 粘贴完成后无浮层提示（用户感知是"双击即粘贴"，无中间步骤）。

### FR-007：无辅助功能权限时降级粘贴流程

**需求**：当应用未获得辅助功能权限时，双击文本类型行走降级粘贴流程。

**行为**：

- 检测辅助功能权限状态（运行时检测，不弹 TCC 提示对话框）。
- 无权限时：写入剪贴板 → 关闭快速粘贴面板 → 显示浮层提示"已复制，按 Cmd+V 粘贴"。
- 浮层提示显示位置：屏幕中央或上次关闭位置。
- 浮层提示消失条件（任一满足即消失）：
  - 监听到剪贴板被消费（剪贴板变化计数减少或被其他应用覆盖）。
  - 超时兜底消失（默认 5 秒，可通过设置项配置）。
- 浮层提示不抢夺前台应用焦点（保持在前台应用之上但不激活）。

### FR-008：回车键粘贴

**需求**：高亮选中某行后，按回车键触发粘贴流程（等同于双击）。

**行为**：

- 高亮选中行 + 按回车：触发与双击相同的粘贴流程（FR-006 或 FR-007）。
- 未高亮选中行 + 按回车：不触发任何操作（或忽略）。

### FR-009：Esc 键关闭

**需求**：按 Esc 键关闭快速粘贴面板，不触发粘贴。

**行为**：

- 按 Esc 键：立即关闭快速粘贴面板。
- 不写入剪贴板、不触发粘贴流程。
- 关闭时记录面板位置（供下次无权限定位使用，FR-002）。

### FR-010：失焦自动关闭

**需求**：快速粘贴面板失去键盘焦点时自动关闭。

**行为**：

- 面板失去键盘焦点（用户点击其他应用、点击桌面、切换空间等）：自动关闭面板。
- 不触发粘贴流程。
- 关闭时记录面板位置（供下次无权限定位使用，FR-002）。

### FR-011：粘贴后自动关闭

**需求**：双击或回车触发粘贴后，快速粘贴面板自动关闭。

**行为**：

- 双击/回车触发粘贴流程后：立即关闭快速粘贴面板。
- 有权限路径：关闭面板 → 模拟粘贴按键（粘贴在面板关闭后执行，避免面板抢焦点）。
- 无权限路径：关闭面板 → 显示浮层提示。

### FR-012：图片/文件路径类型双击提示

**需求**：双击图片或文件路径类型的列表行时，显示提示"仅支持文本粘贴"，不进入粘贴流程。

**行为**：

- 双击图片类型行：显示提示"仅支持文本粘贴"，不写入剪贴板、不关闭面板。
- 双击文件路径类型行：显示提示"仅支持文本粘贴"，不写入剪贴板、不关闭面板。
- 提示以非阻塞方式显示（如行内提示或浮层提示），用户可继续操作其他行。

### FR-013：辅助功能权限运行时检测与降级

**需求**：在快速粘贴面板触发粘贴流程时，运行时检测辅助功能权限状态，权限被撤销时自动降级。

**行为**：

- 每次触发粘贴流程时检测权限状态（不缓存上一次结果）。
- 权限已授予：走 FR-006 自动粘贴路径。
- 权限未授予：走 FR-007 降级路径。
- 权限在面板打开期间被撤销：下一次双击/回车自动走降级路径。

### FR-014：浮层超时兜底时长可配置

**需求**：浮层提示的超时兜底时长可通过应用设置项配置。

**行为**：

- 新增设置项：浮层超时兜底时长，默认值 5 秒。
- 用户可在设置面板调整时长（范围 1~30 秒，整数或小数）。
- 设置变更立即生效，不需要重启应用。

## 6. Non-functional Requirements（非功能需求）

### NFR-001：响应性能

- 全局快捷键触发到快速粘贴面板出现：≤ 200ms（用户感知"瞬间出现"）。
- 双击/回车触发粘贴到剪贴板写入完成：≤ 100ms。
- 有权限路径：模拟粘贴按键到前台应用插入文本：≤ 300ms（受前台应用响应速度影响，本指标仅约束 ClipMind 侧）。
- 列表行单击高亮、方向键导航：≤ 16ms（60FPS，无肉眼可感延迟）。

### NFR-002：稳定性

- 快速粘贴面板的呼出/关闭/粘贴流程在 100 次连续操作中无崩溃、无内存泄漏（内存增长 < 5MB）。
- 失焦自动关闭、Esc 关闭、粘贴后关闭三种关闭路径互不冲突，不出现"关不掉"或"重复关闭"。
- 辅助功能权限检测在权限被授予/撤销/重新授予的循环中状态正确，不出现"卡在某一路径"。

### NFR-003：安全性

- 不在日志中输出剪贴板原文、密码、Token、验证码等敏感信息。
- 模拟粘贴按键仅发送系统标准粘贴按键事件，不发送任意按键序列。
- 浮层提示不显示剪贴板原文，仅显示"已复制，按 Cmd+V 粘贴"通用文案。
- 写入剪贴板的内容不绕过现有敏感识别与应用黑名单（若内容来自敏感来源，按现有隐私保护流程处理）。

### NFR-004：兼容性

- 兼容 macOS 12.4+（与 F1 主设计规范一致）。
- 兼容 Apple Silicon（arm64）与 Intel（x86_64）。
- 兼容现有菜单栏 StatusItem toggle popover 行为（不破坏）。
- 兼容现有快捷键配置（用户自定义的快捷键仍然有效，只是触发行为改变）。
- 老用户升级无感知迁移：快捷键不变，行为从"唤起主窗口"升级为"呼出快速粘贴面板"。

### NFR-005：可访问性

- 快速粘贴面板的所有交互元素支持 VoiceOver 朗读。
- 键盘导航完整覆盖所有操作（不依赖鼠标）。
- 高亮选中的视觉差异对色盲用户可识别（不仅依赖颜色，辅以形状/边框）。

### NFR-006：合规性（App Store）

- 主 Scheme 保持 App Store 合规：启用 App Sandbox、仅申请最小必要 entitlement、不使用私有 API。
- 无权限降级方案（FR-007）纯沙盒内实现，完全合规。
- 有权限方案（FR-006）使用公开 API + 标准 TCC 权限流程，合规风险独立标注（详见设计文档风险章节）。
- 若 App Store 审核拒绝有权限方案，则该方案暂存到开发验证 Scheme，主 Scheme 仅保留无权限降级方案。

## 7. Constraints（约束）

### 7.1 业务约束

- **不改动主窗口功能**：主窗口入口改为菜单栏点击或"查看全部"按钮，主窗口内的搜索/详情/处理/设置功能不变。
- **保留菜单栏 StatusItem toggle 行为**：菜单栏点击仍弹出原 popover（与快速粘贴面板是两个独立入口）。
- **不引入新的数据模型或持久化方案**：沿用 F1 主设计规范的剪贴板捕获与加密存储基础设施。
- **仅支持文本类型自动粘贴**：图片/文件路径双击提示"仅支持文本粘贴"，不支持自动粘贴。
- **保持快捷键配置能力**：用户仍可在设置中修改全局快捷键，F1.9 不修改快捷键配置 UI。

### 7.2 兼容约束

- **macOS 12.4+**：与 F1 主设计规范一致，不使用 macOS 13+ 专有 API（如无特殊必要）。
- **老用户无感知迁移**：快捷键不变，行为升级，不要求用户重新配置。

### 7.3 合规约束

- **App Store 主 Scheme 合规**：所有面向发布的代码必须满足 App Store Review Guidelines 与 macOS 沙盒、隐私、权限约束。
- **辅助功能权限依赖标注**：有权限方案（FR-006）依赖辅助功能权限，需在设计文档风险章节标注「合规待定」状态。

## 8. Acceptance Criteria（验收标准）

> **格式约定**：每条 AC 用 Given/When/Then 描述，验证方式必须包含测试框架（XCTest/XCUITest/手动）。
> **AC 编号体系**：AC-F1.9-1 ~ AC-F1.9-12（独立于 F1 主规范的 AC-01~AC-26）。

### AC-F1.9-1：全局快捷键呼出快速粘贴面板（非主窗口）

- **Given** 用户已完成首次启动引导，应用处于菜单栏常驻状态
- **When** 用户在任意应用中按下默认全局快捷键
- **Then** 快速粘贴面板窗口出现（独立窗口，不依赖菜单栏按钮位置）；主窗口不被唤起
- **验证方式**：XCUITest + 手动

### AC-F1.9-2：有辅助功能权限时面板显示在 caret 位置附近

- **Given** 应用已获得辅助功能权限，用户在备忘录中点击光标定位 caret
- **When** 用户按下全局快捷键
- **Then** 快速粘贴面板出现在 caret 位置附近（距离 caret 不超过合理偏移，不遮挡 caret）
- **验证方式**：手动 + 截图（受辅助功能 API 限制，自动化测试仅验证面板出现，caret 定位需手动验证）

### AC-F1.9-3：无辅助功能权限时面板显示在屏幕中央或上次关闭位置

- **Given** 应用未获得辅助功能权限
- **When** 用户按下全局快捷键
- **Then** 快速粘贴面板出现在屏幕中央；若上次关闭时记录的位置仍在屏幕可视范围内，则出现在上次关闭位置
- **验证方式**：XCUITest + 手动

### AC-F1.9-4：列表行单击高亮选中，方向键可移动选中

- **Given** 快速粘贴面板已打开，列表有至少 3 行
- **When** 用户单击第二行
- **Then** 第二行高亮显示，其他行取消高亮
- **And When** 用户按下方向键下
- **Then** 第三行高亮，第二行取消高亮
- **And When** 用户按下方向键上
- **Then** 第二行高亮，第三行取消高亮
- **验证方式**：XCUITest

### AC-F1.9-5：列表行双击或回车键触发粘贴流程

- **Given** 快速粘贴面板已打开，列表第一行是文本类型且已高亮选中
- **When** 用户双击第一行，或按回车键
- **Then** 触发粘贴流程（进入 AC-F1.9-6 或 AC-F1.9-7 分支）
- **验证方式**：XCUITest

### AC-F1.9-6：有权限时双击自动粘贴

- **Given** 应用已获得辅助功能权限，用户在备忘录中点击光标定位 caret，快速粘贴面板已打开
- **When** 用户双击列表中某文本类型行
- **Then** 该行文本被写入剪贴板 → 快速粘贴面板关闭 → 系统粘贴按键被模拟发送到前台应用 → 备忘录 caret 位置插入该文本
- **验证方式**：手动 + 录屏（模拟粘贴按键到外部应用需真实环境验证，XCUITest 仅验证剪贴板写入与面板关闭）

### AC-F1.9-7：无权限时双击降级粘贴流程

- **Given** 应用未获得辅助功能权限，快速粘贴面板已打开
- **When** 用户双击列表中某文本类型行
- **Then** 该行文本被写入剪贴板 → 快速粘贴面板关闭 → 浮层提示"已复制，按 Cmd+V 粘贴"出现 → 监听剪贴板被消费后浮层消失，或超时兜底（默认 5 秒）后浮层消失
- **And When** 用户在设置中将浮层超时兜底时长调整为 10 秒，再次触发降级粘贴流程
- **Then** 浮层提示在 10 秒后超时消失（配置变更立即生效，无需重启应用）
- **验证方式**：XCUITest + 手动

### AC-F1.9-8：Esc 键关闭面板，不粘贴

- **Given** 快速粘贴面板已打开
- **When** 用户按下 Esc 键
- **Then** 快速粘贴面板立即关闭；剪贴板内容不变；不触发粘贴流程
- **验证方式**：XCUITest

### AC-F1.9-9：面板失焦自动关闭

- **Given** 快速粘贴面板已打开
- **When** 用户点击其他应用窗口或桌面（面板失去键盘焦点）
- **Then** 快速粘贴面板自动关闭；不触发粘贴流程
- **验证方式**：XCUITest

### AC-F1.9-10：粘贴后面板自动关闭

- **Given** 快速粘贴面板已打开，用户双击或回车触发粘贴
- **When** 粘贴流程启动
- **Then** 快速粘贴面板自动关闭（在有权限路径中，面板先关闭再模拟粘贴按键；在无权限路径中，面板关闭后浮层提示出现）
- **验证方式**：XCUITest

### AC-F1.9-11：图片/文件路径类型双击提示"仅支持文本粘贴"

- **Given** 快速粘贴面板已打开，列表中包含图片类型行或文件路径类型行
- **When** 用户双击图片类型行或文件路径类型行
- **Then** 显示提示"仅支持文本粘贴"；不写入剪贴板；不关闭面板；用户可继续操作其他行
- **验证方式**：XCUITest

### AC-F1.9-12：辅助功能权限被撤销时自动降级

- **Given** 应用曾获得辅助功能权限，用户在系统设置中撤销了该权限
- **When** 用户下次触发粘贴流程（双击或回车）
- **Then** 自动走无权限降级路径（AC-F1.9-7），不尝试模拟粘贴按键
- **验证方式**：手动（权限撤销需在系统设置中操作，自动化测试仅验证降级逻辑的正确性）

## 9. Out of Scope（明确不做）

### 9.1 不改主窗口功能

- 主窗口内的搜索、详情面板、一键处理、设置功能不变。
- 主窗口入口改为菜单栏点击或"查看全部"按钮（原快捷键入口改归快速粘贴面板）。
- 不调整主窗口布局、视觉、交互。

### 9.2 不支持图片/文件路径自动粘贴

- 图片类型双击仅显示"仅支持文本粘贴"提示，不写入剪贴板、不粘贴。
- 文件路径类型双击仅显示"仅支持文本粘贴"提示，不写入剪贴板、不粘贴。
- 后续迭代再考虑图片/文件路径的粘贴方案。

### 9.3 不实现多选批量粘贴

- 列表行仅支持单选（高亮选中最多一行）。
- 不支持 Shift+点击多选、Cmd+点击多选。
- 不支持"选中多行后一次性粘贴合并文本"。

### 9.4 不修改全局快捷键配置 UI

- 保留现有快捷键录制器（设置面板中的快捷键配置入口）。
- 用户仍可自定义快捷键，F1.9 不修改配置 UI。
- F1.9 仅改变快捷键的触发行为（从唤起主窗口改为呼出快速粘贴面板）。

### 9.5 不实现二次快捷键 toggle 关闭

- 快捷键是单向触发（按一次呼出面板）。
- 不支持"再按一次关闭面板"（关闭通过 Esc/失焦/粘贴完成）。
- 避免快捷键在面板已打开时被前台应用拦截导致状态不一致。

### 9.6 不引入新的数据模型或持久化方案

- 不新增剪贴板条目字段（不增加"已粘贴次数"等统计字段）。
- 不新增持久化表（沿用现有加密 SQLite）。
- 不修改现有 ClipItem 数据模型。

### 9.7 不修改菜单栏 StatusItem toggle 行为

- 菜单栏点击仍弹出原 popover（与快速粘贴面板是两个独立入口）。
- 不删除原 popover，不合并原 popover 与快速粘贴面板。

## 10. Terminology（术语）

| 术语 | 定义 |
|------|------|
| 快速粘贴面板 | F1.9 新增的独立窗口，由全局快捷键呼出，显示历史剪贴项列表，支持单击选中、双击粘贴、键盘导航 |
| 全局快捷键 | 用户可在任意应用中按下的系统级快捷键，由应用通过系统 API 注册，默认值与现有配置一致 |
| 前台应用 | 当前拥有键盘焦点的应用，即用户准备粘贴内容的目标应用 |
| caret 位置 | 前台应用中文本输入光标的位置，用于定位快速粘贴面板 |
| 辅助功能权限 | macOS 系统的辅助功能 TCC 权限，授予后应用可获取前台应用 caret 位置、模拟按键事件 |
| 自动粘贴 | 有辅助功能权限时的粘贴路径：写入剪贴板 → 模拟系统粘贴按键 → 前台应用插入文本 |
| 降级粘贴 | 无辅助功能权限时的粘贴路径：写入剪贴板 → 显示浮层提示 → 用户手动按 Cmd+V 粘贴 |
| 浮层提示 | 降级粘贴路径中显示的轻量级提示窗口，文案为"已复制，按 Cmd+V 粘贴"，不抢夺前台应用焦点 |
| 剪贴板消费 | 剪贴板内容被其他应用读取或覆盖（变化计数变化），作为浮层提示消失的条件之一 |
| 超时兜底 | 浮层提示显示的最大时长（默认 5 秒），超时后强制消失，避免浮层长期停留 |
| 菜单栏 StatusItem | macOS 系统菜单栏的应用图标，点击弹出原 popover（与快速粘贴面板是两个独立入口） |
| 主窗口 | ClipMind 的完整功能窗口，包含历史列表、搜索、详情、处理、设置，通过菜单栏点击或"查看全部"按钮进入 |
| 文本类型行 | 列表中内容为纯文本的行，支持自动粘贴 |
| 图片/文件路径类型行 | 列表中内容为图片或文件路径的行，不支持自动粘贴，双击仅显示提示 |

## 11. Decision Freedom（实现自由度）

### 11.1 允许的实现自由度

- **窗口类型选择**：快速粘贴面板可以是系统提供的浮动窗口、标准窗口或弹出式窗口的任意一种，只要满足"独立窗口、可定位到任意屏幕坐标、失焦自动关闭"三个约束。
- **窗口层级选择**：快速粘贴面板可使用系统提供的窗口层级（如浮动窗口层级），只要保证在前台应用之上且不抢夺焦点。
- **caret 位置获取方式**：有权限路径中，可通过系统辅助功能 API 获取前台应用 caret 位置，或通过其他公开 API（如系统文本输入上下文）获取，只要不使用私有 API。
- **模拟粘贴按键方式**：有权限路径中，可通过系统事件 API 模拟系统粘贴按键，或通过其他公开 API 触发粘贴，只要不发送任意按键序列。
- **剪贴板消费监听方式**：降级路径中，可通过轮询剪贴板变化计数或注册剪贴板监听通知来检测消费，只要不依赖私有 API。
- **浮层提示窗口类型**：浮层提示可以是系统提供的浮动窗口、标准窗口或自定义视图的任意一种，只要满足"不抢夺前台应用焦点、超时消失、消费后消失"三个约束。
- **键盘导航实现方式**：方向键导航可通过声明式 UI 框架的按键事件修饰符、应用工具包的按键事件回调或其他公开 API 实现，只要满足"上下方向键移动高亮、回车粘贴、Esc 关闭"约束。
- **列表行视图复用方式**：可复用现有列表行视图骨架并扩展，或新建独立列表行视图，只要视觉与现有 popover 列表行一致。
- **设置项存储位置**：浮层超时兜底时长可存储在现有应用设置中，或独立存储，只要满足"用户可配置、变更立即生效"约束。

### 11.2 禁止的实现方式

- **禁止使用私有 API**：所有实现必须使用公开 API，满足 App Store 合规。
- **禁止绕过沙盒**：所有实现必须在 App Sandbox 内完成，不通过外部进程、脚本或辅助工具绕过沙盒限制。
- **禁止缓存辅助功能权限状态**：每次触发粘贴流程时必须运行时检测权限状态，不缓存上一次结果（避免权限被撤销后仍走有权限路径）。
- **禁止弹 TCC 提示对话框**：快速粘贴面板触发粘贴流程时不弹 TCC 提示（避免打断用户工作流），仅查询当前权限状态。
- **禁止在日志中输出剪贴板原文**：所有日志仅记录元数据（如"已写入剪贴板，长度 256"），不记录剪贴板原文。
- **禁止发送任意按键序列**：模拟粘贴按键仅发送系统标准粘贴按键，不发送其他按键。
- **禁止浮层提示显示剪贴板原文**：浮层提示仅显示"已复制，按 Cmd+V 粘贴"通用文案。
- **禁止修改主窗口功能**：主窗口内的搜索/详情/处理/设置功能不变。
- **禁止删除菜单栏原 popover**：菜单栏 StatusItem toggle 行为保留，与快速粘贴面板是两个独立入口。
- **禁止使用固定 sleep 等待异步逻辑**：浮层提示的消失条件（消费/超时）必须使用明确的条件等待，不用固定 sleep 掩盖竞态。

## 12. Future Considerations（未来扩展）

### 12.1 图片/文件路径自动粘贴

- 当前仅支持文本类型自动粘贴，未来可扩展图片类型（写入剪贴板图片 + 模拟粘贴）与文件路径类型（写入剪贴板文件 URL + 模拟粘贴）。
- 需考虑前台应用是否支持图片/文件粘贴（如备忘录支持图片、终端不支持图片）。

### 12.2 多选批量粘贴

- 当前仅支持单选，未来可扩展 Shift+点击多选、Cmd+点击多选。
- 多选后可"选中多行后一次性粘贴合并文本"（按时间顺序或选中顺序合并）。

### 12.3 二次快捷键 toggle 关闭

- 当前快捷键是单向触发，未来可扩展"再按一次关闭面板"。
- 需解决快捷键在面板已打开时被前台应用拦截的状态一致性问题。

### 12.4 粘贴历史统计

- 当前不记录粘贴次数，未来可扩展"已粘贴次数"统计字段，用于排序（最常粘贴优先）。
- 需考虑隐私：粘贴次数是元数据，不涉及剪贴板原文，可安全存储。

### 12.5 智能排序

- 当前列表按时间倒序，未来可扩展智能排序（最近粘贴优先、最常粘贴优先、当前前台应用相关优先）。
- 需考虑性能：智能排序需在面板出现前完成，避免延迟。

### 12.6 自定义面板尺寸

- 当前面板尺寸固定，未来可扩展用户自定义尺寸（拖拽调整、设置项配置）。
- 需考虑记忆用户上次尺寸偏好。

### 12.7 多显示器支持

- 当前面板定位在主屏幕，未来可扩展多显示器支持（面板出现在光标所在显示器）。
- 需考虑跨显示器的坐标转换。
