# F1.9 快捷粘贴面板 — 步骤 0 需求摘要

> 本文件由 dd-feature-development-workflow 步骤 0.4 写入，供步骤 2 调用 dd-writing-specs 时复用，跳过重复 grill。
> 清理责任归本工作流步骤 9.1，dd-writing-specs 不清理此文件。

## 1. 用户问题和业务目标

**问题**：当前剪贴板粘贴需切换到主窗口或菜单栏 Popover（位置绑定菜单栏），用户在任意应用准备粘贴时操作路径过长。

**目标**：用户在任意应用准备粘贴时，按全局快捷键直接在粘贴目标附近呼出快速粘贴面板，单击选中、双击粘贴，无需切换应用上下文。

## 2. 成功标准

- 用户在任意应用中按 `cmd+shift+v`，在 caret/鼠标附近出现 Popover 窗口 A
- 列表显示历史剪贴项，单击高亮选中、双击粘贴到前台应用
- 有 Accessibility 权限时自动粘贴；无权限时写入剪贴板并显示浮层提示
- 浮层提示在剪贴板被消费后消失，或 5s 超时消失
- 所有 AC 通过 XCTest/XCUITest + 手动验收验证

## 3. 范围边界

### 必须做

- 改造 `GlobalHotkeyService`：`cmd+shift+v` 改为呼出 Popover 窗口 A（不再唤起主窗口）
- 新建独立 `QuickPasteWindow/Panel`（不依赖 StatusItem button），可定位到任意屏幕坐标
- 运行时检测 Accessibility 权限：有权限走 caret 定位 + 自动 Cmd+V；无权限走屏幕中央/上次位置 + 浮层提示
- `ClipRowView` 增加单击选中、双击粘贴交互
- 键盘导航：方向键上下移动选中、回车粘贴、Esc 关闭
- 失焦自动关闭、粘贴后自动关闭

### 明确不做

- 不改动主窗口功能（主窗口入口改为菜单栏点击或"查看全部"按钮）
- 不支持图片/文件路径类型的自动粘贴（显示但双击提示"仅支持文本"，后续迭代）
- 不实现多选批量粘贴
- 不修改全局快捷键配置 UI（保留现有 HotkeyRecorder）
- 不实现二次快捷键 toggle 关闭（保持单向触发）

## 4. 用户流程

- **入口**：用户在任意应用准备粘贴时按 `cmd+shift+v`
- **主要路径**：
  1. App 检测 Accessibility 权限
  2. 有权限：获取前台应用 caret 位置，Popover 显示在 caret 附近
  3. 无权限：Popover 显示在屏幕中央或上次关闭位置
  4. 用户在 Popover 中搜索/浏览列表
  5. 单击高亮选中某行
  6. 双击该行：
     - 有权限：写入剪贴板 → 关闭 Popover → 模拟 Cmd+V 到前台应用
     - 无权限：写入剪贴板 → 关闭 Popover → 显示浮层提示"已复制，按 Cmd+V 粘贴" → 监听剪贴板被消费后消失，或 5s 超时消失
- **失败路径**：
  - Accessibility 权限被撤销：降级到无权限方案
  - 前台应用无 caret（如 Finder）：降级到鼠标位置（NSEvent.mouseLocation）
  - 剪贴板写入失败：错误日志 + 浮层提示"粘贴失败"
- **退出条件**：Esc / 失焦 / 粘贴完成

## 5. 数据和接口

### 新增

- `QuickPastePanelController`：独立窗口控制器
- `AccessibilityService`：封装 AXUIElement，检测权限、获取 caret 位置
- `PasteSimulator`：封装 CGEvent 模拟 Cmd+V
- `PasteOverlayController`：无权限降级浮层
- 通知名 `.openQuickPaste`（替代 `.openMainWindow` 用于快捷键路径）

### 修改

- `GlobalHotkeyService.handleHotkeyPressed()`：改发 `.openQuickPaste`
- `ClipRowView`：增加 `isSelected`、单击/双击手势
- `PopoverView`：保留为菜单栏 Popover；新建 `QuickPasteView` 复用搜索框+列表+底部按钮

### 配置

- `AppSettings` 新增 `quickPasteOverlayDuration: Double = 5.0`（浮层超时兜底时长）

## 6. 兼容和迁移

- 现有 `cmd+shift+v` 默认行为从"唤起主窗口"改为"呼出 Popover 窗口 A"
- 主窗口入口：菜单栏点击 → Popover"查看全部"按钮 → 主窗口
- 不影响现有快捷键配置（用户仍可在设置中修改 hotkey）
- 保留菜单栏 StatusItem 的 toggle Popover 行为
- 老用户升级无感知迁移（hotkey 不变，行为升级）

## 8. 阶段拆分

| Phase | 目标 | 风险 |
|-------|------|------|
| Phase 1 | 快捷键改呼出独立 Popover 窗口 A + 屏幕中央/上次位置定位（合规基线） | 低 |
| Phase 2 | ClipRowView 单击选中 + 双击粘贴 + 键盘导航 | 中 |
| Phase 3 | 无权限降级粘贴流程（写剪贴板 + 浮层提示 + 监听 + 5s 超时） | 中 |
| Phase 4 | Accessibility 权限路径（caret 定位 + 自动 Cmd+V）— 独立 phase 便于回退 | 高（合规） |
