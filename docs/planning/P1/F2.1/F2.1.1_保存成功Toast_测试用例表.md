| 手动测试 | 验证无法自动化的场景 | 动画视觉效果录屏、源 App 全屏遮挡、TCC 弹窗验证、性能 Instruments 验证 | 真实 Safari/Notes 复制时观察 Toast 行为，Instruments 录制帧率与内存 |

**测试集存放**：
- 单元测试：`ClipMindTests/`
- UI 测试：`ClipMindUITests/`
- 测试 Fixtures：`ClipMindTests/Fixtures/`

**异步等待策略**（D7 落地，参照 F2.1 D17 可测试性原则）：
- ❌ 禁止使用 `sleep 3` 等固定 sleep 等待异步结果
- ✅ 改为轮询目标条件 + 超时 3s + 条件满足即继续
- ✅ 轮询工具：`PollingHelper.waitUntil(condition:timeout:)`（10ms 轮询间隔，3s 超时）
- ✅ 2 秒计时与 0.2 秒动画通过 D7 注入虚拟计时器源加速测试，不依赖真实时间等待
- ✅ 动画时长作为可注入参数（默认 0.2 秒，测试时可缩短为 0 秒验证状态转换不依赖动画时长）
- **XCUITest 工具选型**：UI 测试中等待 Toast 出现/消失使用 `XCUIElement.waitForExistence(timeout:)` 或 `NSPredicate` + `XCTestExpectation` 轮询 toast-container 存在/不存在条件，禁用 `Thread.sleep`；2 秒显示计时与 0.2 秒动画在单元测试中通过 D7 注入的虚拟计时器源加速推进，在 XCUITest 中等待实际时长（约 2.5 秒含余量，使用 `waitForExistence(timeout:)` 轮询条件而非固定 sleep）

## 8. 测试策略

### 8.1 单元测试与集成测试策略（XCTest，D7 落地）

- **测试目标**：验证设计文档 v1.1 中 3 个新增模块的职责边界与内部逻辑，覆盖 5 状态机的 8 个转换、替换模式的 5 种状态、7 个错误场景、7 个 D 决策
- **测试组织**：按模块分文件存放于 `ClipMindTests/` 目录，具体文件名由实现规划层决定
- **依赖注入模式**（D4 + D7 落地）：
  - 总开关查询：注入 Mock 闭包（返回启用/禁用/抛错/不可用），不依赖真实 F2.1 配置模块
  - 计时器源：注入虚拟计时器源（可手动推进时间），不依赖真实 2 秒等待
  - 动画时长：作为可注入常量参数（默认 0.2 秒，测试时缩短为 0 秒），验证状态转换不依赖动画时长
  - 屏幕信息：注入 Mock 屏幕尺寸，不依赖真实显示器
- **通知载荷注入**：构造测试通知（含事件标识、文件名、跳过标记）直接派发到协调模块，验证载荷解析与降级
- **Mock 策略**：仅 mock 外部依赖（F2.1 自动保存服务通知、F2.1 配置模块、F1.x 主屏幕信息模块、F1.x 日志模块、窗口创建），不 mock 被测对象本身
- **异步等待**（D7，参照 F2.1 D17）：禁止使用 `sleep 3`，改为 `PollingHelper.waitUntil(condition:timeout:)`（10ms 轮询间隔，3s 超时）；2 秒计时与 0.2 秒动画通过虚拟计时器源加速
- **状态可观测**：协调模块对外暴露当前状态只读属性（或状态变更回调），测试可断言中途态（替换中、消失中）的转换
- **覆盖率目标**：每个新增模块的单元测试覆盖率 ≥ 90%

### 8.2 UI 测试策略（XCUITest）

- **测试目标**：验证用户可见的端到端 UI 交互，覆盖 UI 可观测性矩阵中的 8 项 UI AC
- **测试组织**：按 AC 分文件存放于 `ClipMindUITests/` 目录，具体文件名由实现规划层决定
- **覆盖范围**：AC-01（Toast 出现）、AC-02（2 秒消失）、AC-03（文件名显示）、AC-04（替换模式）、AC-08（动画不阻塞显示）、AC-09（顶部居中位置）、AC-10（Sandbox 合规）、AC-11（不依赖焦点）
- **真实入口**：使用真实 ClipMind App 启动，模拟用户在白名单 App 中复制操作
- **accessibility identifier 定位**：
  - `toast-container`：断言 Toast 存在/不存在（对应 AC-01、AC-02、AC-04、AC-11）
  - `toast-filename-text`：断言文件名文本与预期一致（对应 AC-03、AC-04）
  - `toast-success-icon`：断言成功图标可见（对应 FR-002）
  - 前缀区分：Toast 容器前缀 `toast-`，F2.1 错误弹窗前缀 `error-`，XCUITest 通过前缀区分避免 AC-07 误判
- **AC-04 替换模式验证**：触发两次保存后立即断言 toast-filename-text 显示新文件名，再等待 2.5 秒断言已消失（验证 2 秒计时从新 Toast 出现时刻重新计算）
- **AC-08 动画验证**：进入动画启动后立即断言 toast-container 存在（不等待 0.2 秒动画完成，验证窗口先创建后启动动画的关键不变量）
- **证据存储**：截图与录屏存放在 `docs/planning/P1/F2.1/screenshots/` 与 `recordings/`

### 8.3 手动测试策略

- **测试目标**：验证无法自动化的场景
- **测试场景**：
  - 动画视觉效果录屏：录屏观察进入动画（滑入 + 淡入）与退出动画（滑出 + 淡出）的视觉效果，验证 60fps 流畅
  - 源 App 全屏遮挡：在 Safari 全屏模式下复制长内容，观察 Toast 是否被全屏 App 遮挡（R-05 风险验证）
  - TCC 弹窗验证：主 Scheme 运行时录屏，确认无 TCC 权限弹窗出现（AC-10）
  - 不依赖焦点验证：在 Safari 前台复制长内容，截图验证 Toast 可见且 Safari 仍处于前台（AC-11）
- **证据存储**：手动测试记录存放在 `docs/planning/P1/F2.1/screenshots/` 与 `recordings/`

### 8.4 性能测试策略

- **测试目标**：验证 NFR-001（≤0.3 秒）、NFR-002（60fps）、NFR-003（资源占用与释放）
- **测试方式**：
  - NFR-001 响应性能：XCTest 性能测试记录通知接收到 Toast 出现的实际耗时，断言 P95 ≤ 0.3 秒（含动画启动时间）
  - NFR-002 动画流畅度：Instruments Core Animation 工具验证进入/退出动画保持 60fps
  - NFR-003 资源占用：Instruments Time Profiler 验证显示期间 CPU 不可感知；Instruments Allocations 验证显示期间内存不可感知，消失后窗口对象立即释放
- **性能测量工具**：`PerformanceMeasurer.measure(block:)` 记录耗时并计算 P95（参照 F2.1 D21）
- **不使用固定 sleep 等待**：性能测试通过虚拟计时器源推进时间，不依赖真实等待
