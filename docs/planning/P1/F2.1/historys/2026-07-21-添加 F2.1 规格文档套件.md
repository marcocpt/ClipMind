## 为什么

F2.1 自动保存到文件功能需要完整的规格文档套件作为实现规划的输入。本工作流遵循 dd-writing-specs 的"先拷问再写规范，三审并行再确认，一份做完再做下一份"原则，确保文档分层清晰（Requirements WHAT / Design WHO / Visual UI / Test Cases HOW 验证），P0 铁律合规（不写代码符号）。

## 影响哪些部分

- **后续实现规划**：基于本套件的 FR/NFR/AC/模块职责编写实现规划（接口签名、类定义、目录结构）
- **后续代码实现**：基于测试用例表的 49 个测试用例编写 XCTest/XCUITest
- **后续 UI 实现**：基于视觉原型实现配置面板自动保存分区
- **F1.x 既有功能**：不影响（F2.1 仅通过自动保存钩子插入检查点，不修改 F1.x 既有公共接口）
