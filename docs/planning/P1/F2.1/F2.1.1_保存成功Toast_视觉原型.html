状态说明面板（需求与验收映射）
列出本视觉原型覆盖的功能需求（FR）与验收标准（AC），便于对照交互行为。
FR-002
显示内容：成功图标（绿色圆形 + 白色对勾）+ 实际保存的文件名（含扩展名）
FR-003
显示位置：主屏幕顶部居中，水平居中，垂直位于屏幕顶部安全区域内
FR-004
显示时长：2 秒后自动消失，消失过程带退出动画
FR-005
进入动画：从顶部滑入（translateY -100% → 0）+ 淡入（opacity 0 → 1），时长 0.2 秒，ease-out 缓动
FR-006
退出动画：反向滑出（translateY 0 → -100%）+ 淡出（opacity 1 → 0），时长 0.2 秒，ease-in 缓动
FR-007
替换模式：新 Toast 出现时立即关闭旧 Toast（无退出动画），新 Toast 触发进入动画，2 秒计时从新 Toast 出现时刻重新计算
FR-009
跳过场景（长度不足、敏感过滤、黑名单、变更计数过期）不显示 Toast
FR-010
失败场景不显示 Toast，仍走 F2.1 既有错误弹窗机制
FR-011
Toast 不依赖 ClipMind 主窗口或菜单栏弹窗的焦点状态，作为屏幕级浮层显示
AC-08
动画验证：进入动画启动后 Toast 容器立即可见（窗口先创建后启动动画，动画不阻塞显示）
AC-09
位置验证：Toast 水平居中，垂直位于屏幕顶部 16-32pt 范围内（本原型取 24px）
AC-01
Toast 出现：保存成功后 Toast 立即出现在屏幕顶部居中。验证方式：XCUITest 触发保存后断言 toast-container 存在。
AC-02
2 秒消失：Toast 显示 2 秒后自动消失（含退出动画）。验证方式：XCUITest 等待 2 秒后断言 toast-container 不存在。
AC-03
文件名显示：Toast 显示实际保存的文件名（含扩展名）。验证方式：XCUITest 断言 toast-filename-text 文本与预期文件名一致。
AC-04
替换模式：新 Toast 出现时立即关闭旧 Toast（无退出动画），2 秒计时重新计算。验证方式：XCUITest 快速连续触发后断言仅存在一个 toast-container。
AC-11
Toast 不依赖窗口焦点：Toast 作为屏幕级透明窗口浮层，不依赖 ClipMind 主窗口或菜单栏弹窗的焦点状态；用户在源 App（浏览器、IDE）前台复制时，Toast 仍能显示在屏幕顶部。验证方式：手动验收截图（源 App 前台 + Toast 可见）+ XCUITest 在源 App 处于前台时触发保存后断言 toast-container 存在。HTML 原型限制：HTML 无法真正模拟源 App 前台，实际行为由 macOS 原生透明窗口保证（对应设计文档 D1 决策）。
AC-06/07/10
AC-06（F2.1 总开关关闭）、AC-07（失败场景，错误弹窗存在 + Toast 不存在）、AC-10（App Sandbox 合规）属于系统级行为，由 XCUITest 与运行录屏验证，不在视觉原型演示范围。
Toast 协调模块状态机（详见设计文档 §5.1）

Accessibility Identifier 命名约定
关键 UI 元素的 accessibility identifier 命名约定，供 XCUITest 断言使用。HTML 原型以 aria-label / role 属性作为示例，实际 SwiftUI 实现时由实现规划层落地为 .accessibilityIdentifier()。
toast-container
Toast 容器。XCUITest 断言 Toast 存在 / 不存在（对应 AC-01、AC-02、AC-04、AC-11）。
toast-filename-text
文件名文字。XCUITest 断言文件名文本与预期一致（对应 AC-03）。
toast-success-icon
成功图标（绿色圆形 + 白色对勾）。XCUITest 断言图标可见（对应 FR-002）。
前缀区分
Toast 容器 identifier 前缀：toast-。F2.1 错误弹窗 identifier 前缀：error-（由 F2.1 既有实现定义，本特性不修改）。XCUITest 通过 identifier 前缀区分两者，避免 AC-07（错误弹窗存在 + Toast 不存在）误判。
