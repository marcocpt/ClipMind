# F2.1 自动保存到文件 需求摘要（步骤 0 产出）

> 最后更新：2026-07-21 | 版本：v1.0

本文档由 dd-feature-development-workflow 步骤 0.4 产出，供步骤 2 调用 dd-writing-specs 时复用（dd-writing-specs 步骤 1.0 检测到本文件存在则跳过 grill，直接复用本摘要）。

## 1. 用户问题与业务目标

**问题**：从 ChatGPT 网页、Trae、VSCode、Xcode 等 App 复制长内容后，用户需要手动新建文件、粘贴、命名、再复制路径才能在 Trae 中通过 `@` 引用，重复劳动消耗专注时间。

**目标**：在白名单 App 中复制长内容时，自动保存到指定目录的文件，并自动复制文件路径到剪贴板，让 Trae 能直接 `@` 引用文件。

## 2. 成功标准

- 在白名单 App 中复制 ≥50 字内容 → 3 秒内自动保存为 .md/.txt 文件
- 剪贴板自动替换为文件绝对路径，原内容仍入库 ClipMind 历史
- 配置面板可配置：保存目录、白名单 App、文件格式、长度阈值、文件名长度
- 路径格式可配置（纯路径 / file:// URI / Markdown 链接）

## 3. 范围边界

### 3.1 必须做

- 白名单 App 触发
- 内容长度阈值过滤（默认 50 字，可配置）
- 文件保存到单一目录（可配置）
- 文件名前缀命名（取内容前 N 字，过滤特殊字符，可配置长度）
- 文件名冲突追加序号（content-1.md、content-2.md）
- 剪贴板替换为文件路径
- 原内容仍入库 ClipMind 历史
- 配置面板新增"自动保存"独立分区
- 总开关（启用/禁用整个功能）
- 敏感内容过滤（默认开，可关闭）

### 3.2 明确不做

- 文件内容编辑
- 文件上传云端
- 跨设备文件同步
- 文件内容 OCR
- 富文本格式保留
- 自定义路径模板（仅单一目录）
- 按 App 分目录

## 4. 用户流程

1. 用户在白名单 App（如 Safari 访问 ChatGPT）复制 ≥50 字内容
2. ClipMind 监听到复制 → 检查总开关 → 检查白名单 → 检查长度阈值 → 敏感过滤（默认开）→ 生成文件名（前 20 字 + 过滤特殊字符）→ 写入文件（冲突追加序号）→ 替换剪贴板为文件路径 → 原内容入库历史
3. 用户切换到 Trae → `@` 触发引用 → 粘贴路径 → 直接引用文件

### 失败路径

- 保存目录不存在/无写权限 → 弹窗提示用户检查配置，不崩溃，不替换剪贴板
- 敏感内容命中（默认开）→ 不保存到文件，原内容仍按现有 F1.x 流程处理（不入库）
- 长度不足 → 静默不触发，原内容按现有流程入库

## 5. 数据和接口

### 5.1 新增模型

- `AutoSaveSettings`：保存目录、白名单 bundleId 列表、文件格式（.md/.txt）、长度阈值、文件名长度、敏感过滤开关、路径格式、总开关
- `AutoSaveRecord`：保存记录（文件路径、原内容摘要、时间戳、来源 App），可用于历史查看

### 5.2 新增服务

- `AutoSaveService`：协调总开关、白名单检查、长度检查、敏感检查、文件写入、剪贴板替换
- `FileNameGenerator`：生成文件名（前缀 + 过滤特殊字符 + 序号）
- `FilePathFormatter`：格式化路径（纯路径 / file:// / Markdown）

### 5.3 修改

- `PasteboardWatcher` 或 `ClipCaptureService`：在捕获流程中插入 AutoSave 钩子
- `AppSettings`：新增 autoSave 字段
- 设置面板：新增"自动保存"独立分区

### 5.4 配置项默认值

| 配置项 | 默认值 |
|--------|--------|
| 总开关 | 开 |
| 保存目录 | `~/Documents/ClipMind/Clips/` |
| 白名单 App | Safari、Google Chrome、Trae IDE、VSCode、Xcode |
| 文件格式 | .md |
| 长度阈值 | 50 字 |
| 文件名长度 | 20 字 |
| 敏感过滤 | 开 |
| 路径格式 | 纯路径字符串 |

## 6. 兼容和迁移

- 新增配置项，不影响现有 F1.x 功能
- 可独立启用/禁用（总开关）
- 老版本升级：首启预置默认配置
- 与 F1.x 剪贴板流程的关系：原内容仍按现有流程入库，AutoSave 仅在剪贴板内容上"额外做一次保存到文件 + 路径替换"

## 7. 验收标准 AC（草案，最终以需求文档为准）

- **AC-01**：白名单 App 复制 ≥50 字内容 → 3 秒内文件出现在配置目录，剪贴板替换为路径
- **AC-02**：非白名单 App 复制 → 不触发保存
- **AC-03**：白名单 App 复制 <50 字内容 → 不触发保存
- **AC-04**：文件名冲突 → 自动追加序号（content-1.md、content-2.md）
- **AC-05**：原内容仍进入 ClipMind 历史（不丢失）
- **AC-06**：敏感内容（Token/密码）默认不保存到文件
- **AC-07**：配置面板可修改保存目录、白名单、格式、阈值、文件名长度
- **AC-08**：禁用总开关后，复制不触发任何保存行为
- **AC-09**：保存目录不存在/无写权限 → 弹窗提示，不崩溃
- **AC-10**：文件名过滤换行/路径分隔符/特殊字符，中文保留
- **AC-11**：路径格式可在纯路径 / file:// / Markdown 之间切换

- 配置面板入口：新增"自动保存"独立分区

- 路径格式：默认纯路径字符串 + 可配置（file:// / Markdown）
