# F2.1.1 保存成功 Toast 提示 需求摘要（步骤 0 产出）

> 最后更新：2026-07-23 | 版本：v1.0

本文档由 dd-feature-development-workflow 步骤 0.4 产出，供步骤 2 调用 dd-writing-specs 时复用（dd-writing-specs 步骤 1.0 检测到本文件存在则跳过 grill，直接复用本摘要）。

---

## 1. 用户问题与业务目标

**问题**：F2.1 自动保存到文件成功后，用户无即时视觉反馈，只能凭经验切换到 Trae 后 `@` 引用，无法确认"文件确实保存成功"。当前仅有 `errorNotification` 在保存失败时弹窗，成功路径完全静默。

**目标**：F2.1 自动保存成功后，在屏幕顶部居中位置弹出轻量 Toast 提示（含图标 + 文件名），2 秒后自动消失，让用户在不干扰当前操作流的情况下确认保存已成功。

## 2. 成功标准

- 在白名单 App 中复制 ≥ 长度阈值的内容触发自动保存成功后，**≤ 0.3 秒**内顶部居中弹出 Toast
- Toast 显示 checkmark 图标 + 实际保存的文件名（如 `已保存 hello-world.md`）
- Toast 显示 **2 秒后**自动消失（带退出动画）
- 短时间内多次保存（如快速复制多次）时，**新 Toast 立即替换旧 Toast**，2s 计时从最新一次开始
- 跟随 F2.1 总开关 `isEnabled`，关闭 F2.1 时 Toast 也不显示
- 不影响 F2.1 既有保存流程、剪贴板替换、文件路径入库等行为

## 3. 范围边界

### 3.1 必须做

- 监听 `AutoSaveService.savedNotification`，仅当 `skipped == false` 时显示 Toast
- 顶部居中浮层（不依赖窗口焦点，覆盖在主屏幕顶部）
- SwiftUI 自绘样式：半透明圆角背景 + checkmark 图标 + 文件名文字
- 进入动画：从顶部滑入 + 淡入（约 0.2s）
- 退出动画：反向滑出 + 淡出（约 0.2s）
- 替换模式：新 Toast 出现时立即关闭旧 Toast，2s 计时重置
- 跟随 F2.1 总开关 `isEnabled`
- XCUITest 验证 Toast 出现 / 文件名文本 / 2s 后消失
- 本地手动验收记录（截图 + 录屏）

### 3.2 明确不做

- 不增加独立的 Toast 开关（跟随 F2.1 总开关）
- 不显示文件完整路径（仅文件名，避免路径泄露）
- 不处理失败场景（失败仍走现有 `errorNotification` 弹窗，不新增失败 Toast）
- 不处理跳过场景（敏感过滤、阈值不足、黑名单、changeCount 过期等 `skipped == true` 情况不弹 Toast）
- 不做通知中心集成（不调用 `UNUserNotificationCenter`，避免污染系统通知中心）
- 不做堆叠 / 队列模式（仅替换）
- 不持久化 Toast 状态（App 重启后无历史 Toast）
- 不修改 F2.1 配置面板 UI（无新增配置项）

## 4. 用户流程

### 4.1 主要路径（成功保存）

1. 用户在白名单 App（如 Chrome、Trae、VSCode）中复制 ≥ 长度阈值的长内容
2. `PasteboardWatcher` 捕获事件 → `AutoSaveService.handle(event:)`
3. 通过白名单 / 阈值 / 敏感过滤检查
4. `performSave` 写入文件成功
5. 剪贴板替换为文件路径
6. `postSavedNotification(eventId:fileName:skipped: false)` 发送通知
7. `ToastManager` 监听到通知 → 解析 `fileName` → 在主屏幕顶部居中显示 Toast（带文件名）
8. 2s 后 Toast 自动消失

### 4.2 替换路径（快速多次保存）

1. 用户在 2s 内再次复制触发保存
2. 旧 Toast 立即触发退出动画
3. 新 Toast 触发进入动画并显示新文件名
4. 2s 计时从新 Toast 出现时刻重新计算

### 4.3 失败路径（不弹 Toast）

1. 保存失败（写入异常、冲突重试耗尽等）→ 走现有 `errorNotification` 弹窗，**不弹 Toast**
2. 跳过保存（敏感内容、阈值不足、changeCount 过期等）→ `skipped == true`，**不弹 Toast**
3. F2.1 总开关关闭 → `handle(event:)` 在最早分支返回，**不弹 Toast**

## 5. 数据和接口

### 5.1 新增模型

- `ToastState`（枚举）：`hidden` / `showing(fileName: String)` / `exiting(fileName: String)`
- `ToastManager`（`@MainActor` 类）：监听 `AutoSaveService.savedNotification`，管理 Toast 状态机
- `ToastView`（SwiftUI `View`）：根据 `ToastState` 渲染浮层 + 动画

### 5.2 修改的接口

- 不修改 `AutoSaveService` 公共接口
- 不修改 `AutoSaveService.savedNotification` 的 `Notification.Name` 与 `userInfo` 结构（复用现有 `eventId` / `fileName` / `skipped` 字段）
- AppDelegate 装配：新增 `ToastManager` 实例化与 `savedNotification` 监听注册

### 5.3 持久化

- 不新增持久化字段
- 不修改 `AutoSaveSettings` / `F2xConfigSnapshot`
- 复用 `F2xConfigSnapshot.isEnabled` 作为 Toast 显示前置条件（在 `ToastManager` 内检查，而非 AutoSaveService 内）

## 6. 兼容和迁移

- **不影响 F2.1 既有行为**：保存、剪贴板替换、文件路径入库、错误弹窗等行为完全保持
- **不影响 F1.x 既有行为**：剪贴板监听、分类、加密存储、隐私保护等完全保持
- **禁用 F2.1 时**：`ToastManager` 监听到 `skipped == true`（因为 `handle` 在最早分支 `skipDebug` 后发送 `skipped: true`）→ 不弹 Toast，行为与无此特性一致
- **老版本升级**：无需数据迁移，无新增 UserDefaults 键，无新增配置项
- **App Sandbox 合规**：Toast 仅在 App 内绘制，不调用系统通知 API，不触发额外权限请求，符合主 Scheme `ClipMind` 上架要求

## 7. 验收标准

### AC-01 自动保存成功后弹出 Toast

**场景**：F2.1 启用 + 白名单 App + 复制 ≥ 阈值内容 + 保存成功  
**预期**：屏幕顶部居中出现 Toast，显示 checkmark 图标 + 实际保存的文件名  
**验证**：XCUITest 断言 Toast 容器存在 + 文件名文本可见

### AC-02 Toast 2 秒后自动消失

**场景**：AC-01 触发后等待 2.5 秒  
**预期**：Toast 已消失（退出动画完成）  
**验证**：XCUITest 等待 2.5s 后断言 Toast 容器不存在

### AC-03 Toast 显示文件名

**场景**：保存的文件名为 `hello-world.md`  
**预期**：Toast 文本包含 `hello-world.md`（具体文案格式由设计文档定，但必须包含完整文件名）  
**验证**：XCUITest 断言 Toast 内 label 文本包含 `hello-world.md`

### AC-04 快速多次保存触发替换

**场景**：在 2s 内连续触发两次保存，文件名分别为 `a.md` 与 `b.md`  
**预期**：第二次保存触发后，Toast 立即切换显示 `b.md`，2s 计时从切换时刻重新计算  
**验证**：XCUITest 触发两次保存后立即断言 Toast 显示 `b.md`，再等待 2.5s 断言 Toast 已消失

### AC-05 跳过场景不弹 Toast

**场景**：F2.1 启用 + 复制 < 阈值内容（如复制 5 个字，阈值为 50）  
**预期**：`skipped == true`，不弹出 Toast  
**验证**：XCUITest 触发短内容复制后断言 Toast 不存在

### AC-06 F2.1 总开关关闭时不弹 Toast

**场景**：F2.1 `isEnabled == false` + 复制 ≥ 阈值内容  
**预期**：`AutoSaveService.handle` 在最早分支返回并 `postSavedNotification(skipped: true)`，Toast 不显示  
**验证**：XCUITest 关闭 F2.1 总开关后复制长内容，断言 Toast 不存在

### AC-07 失败场景不弹 Toast（走现有错误弹窗）

**场景**：保存目录不可写导致写入失败  
**预期**：触发 `errorNotification` 弹窗（既有行为），**不**触发 Toast  
**验证**：XCUITest 断言 Toast 不存在 + 错误弹窗存在（或日志 marker 验证 errorNotification 已发送）

### AC-08 进入/退出动画存在

**场景**：保存成功触发 Toast  
**预期**：进入时从顶部滑入 + 淡入（约 0.2s）；退出时反向滑出 + 淡出（约 0.2s）  
**验证**：手动验收录屏 + XCUITest 验证 Toast 在动画时长内可见（动画期间不能断言不存在）

### AC-09 Toast 位置在屏幕顶部居中

**场景**：保存成功触发 Toast  
**预期**：Toast 水平居中，垂直位于主屏幕顶部（距顶部边距由设计文档定，建议 16-32pt）  
**验证**：手动验收截图 + XCUITest 断言 Toast 容器的 frame 水平居中且 frame.minY 在屏幕顶部 0-50pt 范围内

### AC-10 App Sandbox 合规

**场景**：在主 Scheme `ClipMind` 启用 App Sandbox 状态下构建运行  
**预期**：Toast 显示与消失正常，不触发额外权限请求，无私有 API 调用  
**验证**：主 Scheme 构建成功 + 运行无 TCC 弹窗 + XCUITest 通过

## 8. 阶段拆分

### Phase 0：ToastManager + ToastView 核心实现（XCTest 验证）

- 实现 `ToastState` 枚举
- 实现 `ToastManager`（`@MainActor`，监听 `savedNotification`，状态机管理）
- 实现 `ToastView`（SwiftUI 浮层 + 进入/退出动画）
- 编写 XCTest 验证状态机：监听通知 → 进入 `showing` 状态 → 2s 后进入 `exiting` → 回到 `hidden`
- 编写 XCTest 验证替换逻辑：`showing` 状态时再次收到通知立即替换 `fileName` 并重置计时
- 编写 XCTest 验证 `skipped == true` 时不进入 `showing` 状态
- 不实际启动 App，XCTest 单元测试通过即可

### Phase 1：AppDelegate 装配 + XCUITest + 手动验收

- AppDelegate 装配：实例化 `ToastManager`，注册 `savedNotification` 监听
- `ToastManager` 持有 `F2xConfigSnapshot` 引用（或通过闭包读取 `isEnabled`），在收到通知后检查 `isEnabled`
- 集成到主窗口或独立 panel：选择合适的方式承载 ToastView（overlay / NSPanel / 透明窗口）
- 编写 XCUITest：AC-01 ~ AC-10 全部用例
- 触发远程 UI Smoke CI（修改 AppDelegate / 主窗口 → 高风险）
- 本地手动验收：截图 + 录屏 + 验收记录
