## 一、第一轮审查发现的问题与自动修复

### 1.1 accessibility identifier 命名约定缺失（复审 C 发现 - 必须修复）

- **问题**：视觉原型未标注 Toast 容器、文件名文字、图标三个关键 UI 元素的 accessibility identifier 命名约定
- **修复**：新增"Accessibility Identifier 命名约定"区块，明确 `toast-container` / `toast-filename-text` / `toast-success-icon` 三个命名；在 Toast 容器 div、文件名 span、图标 SVG 上添加 `aria-label` / `role="img"` / `role="status"` / `aria-live="polite"` 等可访问性属性作为示例

### 1.2 Toast 与 F2.1 错误弹窗区分方式未说明（复审 C 发现 - 必须修复）

- **问题**：AC-07 验证需要区分 Toast 与错误弹窗，但视觉原型未说明区分方式
- **修复**：在 Accessibility Identifier 命名约定区块中补充前缀区分说明：Toast 用 `toast-` 前缀，F2.1 错误弹窗用 `error-` 前缀（由 F2.1 既有实现定义，本特性不修改），XCUITest 通过前缀避免 AC-07 误判

### 1.3 AC-11 演示说明不够显式（复审 C 发现 - 必须修复）

- **问题**：未在交互演示区或状态说明面板中显式标注 AC-11 核心行为
- **修复**：状态说明面板新增 AC-11 行，说明 Toast 作为屏幕级透明窗口浮层不依赖窗口焦点；验证方式（手动截图 + XCUITest）；HTML 原型限制说明（无法真正模拟源 App 前台，实际行为由 macOS 原生透明窗口保证，对应设计文档 D1 决策）

### 1.4 状态说明面板 AC 覆盖映射不全（复审 A 发现 - 非阻塞）

- **问题**：仅列 AC-08/09，未列 AC-01/02/03/04/11
- **修复**：补齐 AC 映射表，覆盖 AC-01（Toast 出现）、AC-02（2 秒消失）、AC-03（文件名）、AC-04（替换模式）、AC-08（动画）、AC-09（位置）、AC-11（不依赖焦点）

### 1.5 事件日志面板标注（复审 B 发现 - 非阻塞）

- **问题**：未说明"演示辅助，不属于 Toast 视觉规范"
- **修复**：在事件日志面板标题下方添加说明："此面板仅为演示辅助，不属于 Toast 视觉规范，实际实现不包含此面板"

### 1.6 状态机可视化改为引用（复审 B 发现 - 非阻塞）

- **问题**：完整复刻设计文档 §5.1，存在信息重复
- **修复**：改为引用设计文档 §5.1，保留 5 状态名列表（隐藏/出现中/已显示/替换中/消失中）供对照，移除完整转换逻辑复述

### 1.7 距顶部 24px 标注可调整性（复审 B 发现 - 非阻塞）

- **问题**：24px 是固定值但 AC-09 允许 16-32pt 范围
- **修复**：在标注旁补充"AC-09 允许 16-32pt 范围，24px 为默认值，可在实现规划层调整"

### 1.8 响应式 CSS 标注（复审 B 发现 - 非阻塞）

- **问题**：响应式 CSS 未标注"仅服务 HTML 预览"
- **修复**：在响应式 CSS 代码注释中补充"响应式仅服务于 HTML 预览，macOS 原生 Toast 透明窗口尺寸固定，无需考虑响应式"

### 1.9 AC-06/AC-07/AC-10 系统级行为说明（复审 C 发现 - 非阻塞）

- **问题**：视觉原型未覆盖 AC-06（F2.1 总开关关闭）、AC-07（失败场景）、AC-10（App Sandbox 合规）
- **修复**：在状态说明面板中增加说明："AC-06（F2.1 总开关关闭）、AC-07（失败场景）、AC-10（App Sandbox 合规）属于系统级行为，由 XCUITest 与运行录屏验证，不在视觉原型演示范围"
