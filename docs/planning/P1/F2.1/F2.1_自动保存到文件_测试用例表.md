## 6. 测试策略

### 6.1 单元测试策略（XCTest）

- **测试目标**：验证设计文档中各模块的职责边界与内部逻辑
- **测试组织**：按模块分文件（如 `WhitelistCheckerTests.swift`、`FileNameGeneratorTests.swift` 等）
- **测试 Fixtures**：测试数据存放在 `ClipMindTests/Fixtures/`
- **Mock 策略**：仅 mock 外部依赖（F1.x 敏感识别模块、F1.x 应用识别模块、F1.x 加密存储模块、文件系统），不 mock 被测对象本身
- **临时目录**：文件写入测试使用临时目录，不依赖真实文件系统状态
- **覆盖率目标**：每个新增模块的单元测试覆盖率 ≥ 90%

### 6.2 UI 测试策略（XCUITest）

- **测试目标**：验证用户可见的端到端行为，覆盖 UI 可观测性矩阵中的所有 UI AC
- **测试组织**：按 AC 分文件（如 `AutoSaveUITests.swift`、`AutoSaveSettingsUITests.swift`）
- **真实入口**：使用真实 ClipMind App 启动，模拟用户操作
- **断言策略**：文件存在性断言（通过文件系统 API）、剪贴板内容断言（通过剪贴板 API）、历史列表断言（通过 UI 元素查询）、弹窗断言（通过 UI 元素查询）
- **并发场景**：UI 测试中模拟用户连续复制，验证并发安全性（AC-04 文件名冲突）
- **证据存储**：截图与录屏存放在 `docs/planning/P1/F2.1/screenshots/` 与 `recordings/`

### 6.3 手动测试策略

- **测试目标**：验证无法自动化的场景（真实剪贴板来源 App 识别、真实文件系统权限、真实磁盘满场景）
- **测试场景**：
  - 在真实 Safari、Google Chrome、Trae IDE、VSCode、Xcode 中复制内容，验证白名单匹配
  - 配置保存目录为真实只读路径，验证文件写入失败处理
  - 模拟磁盘满场景，验证文件写入失败处理
- **证据存储**：手动测试记录存放在 `docs/planning/P1/F2.1/screenshots/` 与 `recordings/`

### 6.4 CI 集成

- 单元测试与 UI 测试在 GitHub Actions CI（`.github/workflows/ci.yml`）中自动运行
- CI 命令：`xcodebuild test -project ClipMind.xcodeproj -scheme ClipMind -destination 'platform=macOS' -configuration Debug ARCHS=arm64 ONLY_ACTIVE_ARCH=YES CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO`
- CI 失败阻断合并

### 6.5 测试覆盖完整性验证

- 每个 FR 至少一个 AC（已在需求文档 v1.0 中验证）
- 每个 AC 至少一个测试用例（本表 4.1 ~ 4.16 节覆盖 AC-01 ~ AC-16）
- 每个模块至少一个单元测试（本表 4.17 节覆盖 12 个新增模块的核心逻辑）
- 状态机的每个转换都有对应单元测试（本表 TC-UT-24 ~ TC-UT-33）
