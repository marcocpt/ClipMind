## 1. Background（背景）

ClipMind 在复赛扩展阶段已实现 F2.1 自动保存到文件功能：用户在白名单 App 中复制长内容时，自动保存为文件到指定目录，并将文件路径写入剪贴板，便于在 Trae、Cursor 等 AI 编程工具中通过 `@` 引用文件路径。

F2.1 的现有反馈机制存在不对称：

- **失败有反馈**：保存失败时（如目录不可写、冲突重试耗尽），通过错误弹窗通知用户。
- **成功无反馈**：保存成功时完全静默，用户只能凭经验切换到目标 App 后尝试 `@` 引用，无法在切换前确认"文件确实保存成功"。

这种不对称导致两个问题：

1. 用户在切换到 Trae 后才发现保存失败，需要切回源 App 重新复制，打断工作流。
2. 用户对自动保存功能缺乏信任（"它真的保存了吗？"），降低功能采用率。

本特性（F2.1.1）的目标是：在 F2.1 自动保存成功后，提供轻量、不打扰的视觉反馈（Toast 提示），让用户在不离开当前操作流的情况下确认保存已成功。

---

## 2. Problem Statement（问题定义）

| 编号 | 现状问题 | 可验证现象 |
|------|---------|-----------|
| P-01 | 自动保存成功后无即时视觉反馈，用户无法确认保存已成功 | 用户在白名单 App 复制长内容后，屏幕上无任何成功提示，需切换到目标 App 验证 |
| P-02 | 用户对自动保存功能缺乏信任，采用率低 | 用户反馈"不确定有没有保存成功"，倾向于手动新建文件而非依赖自动保存 |
| P-03 | 现有错误弹窗与成功路径反馈不对称，体验不一致 | 失败有弹窗、成功无提示，用户感知"只有出错时才有反应" |

---

## 3. Goals（目标）

### 3.1 用户可见目标

- 自动保存成功后，屏幕顶部居中弹出轻量 Toast 提示，显示已保存的文件名
- Toast 显示 2 秒后自动消失，不干扰用户当前操作
- 短时间内多次保存时，Toast 立即切换显示最新文件名

### 3.2 兼容性目标

- 不影响 F2.1 既有保存流程、剪贴板替换、文件路径入库等行为
- 不影响 F1.x 既有剪贴板监听、分类、加密存储、隐私保护等行为
- 跟随 F2.1 总开关：F2.1 禁用时 Toast 也不显示
- 老版本升级无需数据迁移

### 3.3 合规性目标

- 符合主 Scheme 上架要求：不触发额外权限请求，不调用系统通知 API，不污染系统通知中心
- 在 App Sandbox 启用状态下正常工作

## 4. Scope（范围）

### 4.1 本次包含

- 监听 F2.1 保存完成通知，仅当保存成功（非跳过、非失败）时显示 Toast
- 顶部居中浮层，半透明圆角背景 + 成功图标 + 文件名文字
- 进入动画（从顶部滑入 + 淡入，约 0.2 秒）
- 退出动画（反向滑出 + 淡出，约 0.2 秒）
- 替换模式：新 Toast 出现时立即关闭旧 Toast，2 秒计时从最新一次开始
- 跟随 F2.1 总开关
- XCUITest 验证 Toast 出现 / 文件名文本 / 2 秒后消失
- 本地手动验收记录（截图 + 录屏）

### 4.2 本次涉及但不属于本特性单独实现

- F2.1 保存完成通知的发送逻辑（已由 F2.1 实现，本特性仅监听）
- F2.1 总开关的状态管理（已由 F2.1 实现，本特性仅读取）
- F2.1 错误弹窗（已由 F2.1 实现，本特性不重复）

## 5. Functional Requirements（功能需求）

### FR-001 Toast 触发条件

系统应在 F2.1 自动保存成功后显示 Toast 提示。仅当保存结果为"成功"（非跳过、非失败）时触发。

### FR-002 Toast 显示内容

Toast 应显示成功图标和实际保存的文件名（包含扩展名），让用户确认保存的具体文件。

### FR-003 Toast 显示位置

Toast 应显示在主屏幕顶部居中位置，水平居中，垂直位于屏幕顶部安全区域内。

### FR-004 Toast 显示时长

Toast 应在显示后 2 秒自动消失。消失过程带退出动画。

### FR-005 Toast 进入动画

Toast 出现时应带进入动画：从顶部滑入 + 透明度淡入，动画时长约 0.2 秒。

### FR-006 Toast 退出动画

Toast 消失时应带退出动画：反向滑出 + 透明度淡出，动画时长约 0.2 秒。

### FR-007 多次保存的替换行为

当 Toast 正在显示时，若再次触发保存成功，新 Toast 应立即替换旧 Toast：旧 Toast 立即触发退出，新 Toast 触发进入动画并显示新文件名，2 秒计时从新 Toast 出现时刻重新计算。

### FR-008 跟随 F2.1 总开关

当 F2.1 总开关关闭时，Toast 不应显示。Toast 显示的前置条件之一是 F2.1 已启用。

### FR-009 跳过场景不显示 Toast

当保存结果为"跳过"（如敏感内容过滤、长度阈值不足、黑名单 App、剪贴板变更计数过期等）时，Toast 不应显示。

### FR-010 失败场景不显示 Toast

当保存结果为"失败"（如写入异常、冲突重试耗尽等）时，Toast 不应显示。失败仍走 F2.1 既有的错误弹窗机制。

### FR-011 Toast 不依赖窗口焦点

Toast 应作为屏幕级浮层显示，不依赖 ClipMind 主窗口或菜单栏弹窗的焦点状态。用户在源 App 中复制时，Toast 能直接显示在屏幕顶部。

---

## 6. Non-functional Requirements（非功能需求）

### NFR-001 响应性能

Toast 从保存成功到出现的延迟应不超过 0.3 秒（含动画启动时间）。

### NFR-002 动画流畅度

进入和退出动画应保持 60fps 流畅，无明显卡顿。

### NFR-003 资源占用

Toast 显示期间不应占用可感知的 CPU 或内存资源。Toast 消失后应立即释放相关资源。

### NFR-004 主线程安全

Toast 状态更新和视图渲染必须在主线程边界内完成，避免跨线程访问 UI 状态。

### NFR-005 日志可检索

Toast 的关键状态变更（出现、替换、消失）应记录带功能标签前缀的日志，便于问题排查。日志不得输出文件完整路径或剪贴板原文。

### NFR-006 App Sandbox 合规

Toast 实现不得调用需要额外 entitlement 的 API，不得触发 TCC 权限弹窗，必须在 App Sandbox 启用状态下正常工作。

---

## 7. Constraints（约束）

### C-01 跟随 F2.1 总开关

Toast 不增加独立配置开关，跟随 F2.1 总开关状态。F2.1 禁用时 Toast 必然不显示。

### C-02 不显示文件完整路径

Toast 仅显示文件名（含扩展名），不显示文件所在目录的完整路径，避免路径信息泄露。

### C-03 不污染系统通知中心

Toast 不得调用系统通知 API（如通知中心），仅在 App 内绘制浮层，避免污染系统通知中心。

### C-04 不修改 F2.1 公共接口

本特性不得修改 F2.1 的公共接口、保存完成通知的名称和载荷结构。仅通过监听现有通知实现。

### C-05 不新增持久化字段

本特性不新增用户偏好配置项、配置项或持久化字段。Toast 状态仅在内存中维护，App 重启后无历史 Toast。

### C-06 合规优先

实现时必须先尝试找到符合 App Sandbox 和上架要求的实现方法。只有在穷尽合规方案仍无法实现时，才允许暂存到开发验证 Scheme，且需在设计文档中标注"合规待定"状态。

## 8. Acceptance Criteria（验收标准）

### AC-01 自动保存成功后弹出 Toast

**Given** F2.1 已启用，且复制源 App 在白名单内，且复制内容长度 ≥ 长度阈值  
**When** 用户复制长内容，触发自动保存成功  
**Then** 屏幕顶部居中出现 Toast，显示成功图标 + 实际保存的文件名  
**验证方式** XCUITest 断言 Toast 容器存在 + 文件名文本可见

### AC-02 Toast 2 秒后自动消失

**Given** AC-01 已触发，Toast 正在显示  
**When** 等待 2.5 秒（含退出动画时长）  
**Then** Toast 已完全消失  
**验证方式** XCUITest 等待 2.5 秒后断言 Toast 容器不存在

### AC-03 Toast 显示文件名

**Given** 保存的文件名为 `hello-world.md`  
**When** Toast 显示  
**Then** Toast 文本包含 `hello-world.md`  
**验证方式** XCUITest 断言 Toast 内文本包含 `hello-world.md`

### AC-04 快速多次保存触发替换

**Given** Toast 正在显示文件名 `a.md`  
**When** 在 2 秒内再次触发保存成功，新文件名为 `b.md`  
**Then** Toast 立即切换显示 `b.md`，2 秒计时从切换时刻重新计算  
**验证方式** XCUITest 触发两次保存后立即断言 Toast 显示 `b.md`，再等待 2.5 秒断言 Toast 已消失

### AC-05 跳过场景不弹 Toast

**Given** F2.1 已启用  
**When** 用户复制内容长度 < 长度阈值（如复制 5 个字，阈值为 50）  
**Then** 保存结果为"跳过"，Toast 不显示  
**验证方式** XCUITest 触发短内容复制后断言 Toast 不存在

### AC-06 F2.1 总开关关闭时不弹 Toast

**Given** F2.1 总开关已关闭  
**When** 用户复制 ≥ 长度阈值的长内容  
**Then** 保存流程在最早分支返回，Toast 不显示  
**验证方式** XCUITest 关闭 F2.1 总开关后复制长内容，断言 Toast 不存在

### AC-07 失败场景不弹 Toast

**Given** 保存目录不可写  
**When** 触发自动保存导致写入失败  
**Then** 触发 F2.1 既有的错误弹窗，Toast 不显示  
**验证方式** XCUITest 断言 Toast 不存在 + 错误弹窗存在（或日志标记验证错误通知已发送）

### AC-08 进入/退出动画存在

**Given** 保存成功触发 Toast  
**When** Toast 出现和消失  
**Then** 进入时从顶部滑入 + 淡入（约 0.2 秒）；退出时反向滑出 + 淡出（约 0.2 秒）  
**验证方式** 手动录屏验证动画视觉效果 + XCUITest 在进入动画启动后立即断言 Toast 容器存在（验证进入动画未阻塞显示）

### AC-09 Toast 位置在屏幕顶部居中

**Given** 保存成功触发 Toast  
**When** Toast 显示  
**Then** Toast 水平居中，垂直位于主屏幕顶部（距顶部边距 16-32pt）  
**验证方式** 手动验收截图 + XCUITest 断言 Toast 容器水平居中且垂直位置在屏幕顶部 0-50pt 范围内

### AC-10 App Sandbox 合规

**Given** 主 Scheme 在 App Sandbox 启用状态下构建运行  
**When** 触发保存成功并显示 Toast  
**Then** Toast 显示与消失正常，不触发额外权限请求，无私有 API 调用  
**验证方式** 主 Scheme 构建成功 + 运行录屏验证无 TCC 弹窗 + XCUITest 通过

### AC-11 Toast 不依赖窗口焦点

**Given** ClipMind 主窗口未激活，用户当前焦点在源 App（如浏览器、IDE）  
**When** 用户在源 App 复制长内容触发自动保存成功  
**Then** Toast 仍能在屏幕顶部居中显示，不依赖 ClipMind 主窗口或菜单栏弹窗的焦点状态  
**验证方式** 手动验收截图（源 App 前台 + Toast 可见）+ XCUITest 在源 App 处于前台时触发保存后断言 Toast 容器存在

## 9. Out of Scope（明确不做）

### O-01 不增加独立的 Toast 开关

Toast 不增加独立配置开关，跟随 F2.1 总开关。用户无法单独关闭 Toast 而保留 F2.1。

### O-02 不显示文件完整路径

Toast 仅显示文件名，不显示文件所在目录的完整路径。

### O-03 不处理失败场景

失败场景仍走 F2.1 既有的错误弹窗机制，本特性不新增失败 Toast。

### O-04 不处理跳过场景

跳过场景（敏感过滤、阈值不足、黑名单、剪贴板变更计数过期等）不弹 Toast。

### O-05 不做通知中心集成

不调用系统通知 API，避免污染系统通知中心。

### O-06 不做堆叠或队列模式

短时间内多次保存仅采用替换模式，不支持堆叠显示或队列排队。

### O-07 不持久化 Toast 状态

App 重启后无历史 Toast 记录。Toast 状态仅在内存中维护。

### O-08 不修改 F2.1 配置面板 UI

F2.1 配置面板不新增任何配置项，UI 无变化。

## 10. Terminology（术语）

| 术语 | 含义 |
|------|------|
| Toast | 屏幕顶部居中短暂显示的轻量提示浮层，自动消失 |
| 自动保存 | F2.1 功能：在白名单 App 复制长内容时自动保存为文件并将路径写入剪贴板 |
| 保存完成通知 | F2.1 在保存流程结束时发送的应用内通知，包含事件标识、文件名、是否跳过等信息 |
| 成功 | 保存结果为成功，即文件已写入且（可选）剪贴板已替换 |
| 跳过 | 保存结果为跳过，因敏感过滤、阈值不足、黑名单、剪贴板变更计数过期等原因未实际保存 |
| 失败 | 保存结果为失败，因写入异常、冲突重试耗尽等原因 |
| 替换模式 | 新 Toast 出现时立即关闭旧 Toast，2 秒计时从最新一次开始 |
| F2.1 总开关 | F2.1 配置中控制整个自动保存功能启用/禁用的开关 |

## 11. Decision Freedom（实现自由度）

### 允许

- Toast 容器的具体承载方式（主窗口 overlay / 独立透明窗口 / 系统级浮层），由设计文档决定
- Toast 视觉细节（背景色、圆角半径、图标样式、字体大小），由视觉原型决定
- Toast 状态机的具体状态命名和转换条件，由设计文档决定
- 动画的具体实现方式（原生动画 API / 自定义插值），由实现规划决定
- 监听保存完成通知的具体注册方式（中心化监听 / 闭包回调），由设计文档决定
- 检查 F2.1 总开关状态的具体方式（读取配置快照 / 闭包查询），由设计文档决定

### 禁止

- 不得修改 F2.1 的公共接口、保存完成通知的名称和载荷结构
- 不得调用需要额外 entitlement 的系统 API
- 不得在日志中输出文件完整路径或剪贴板原文
- 不得使用固定 sleep 等待代替明确条件等待
- 不得跨线程访问 UI 状态

## 12. Future Considerations（未来扩展）

### FC-01 独立 Toast 开关

未来可考虑在 F2.1 配置面板新增独立 Toast 开关，让用户单独关闭 Toast 而保留自动保存功能。

### FC-02 多种 Toast 类型

未来可考虑扩展支持多种 Toast 类型（如跳过提示、文件路径复制成功提示等），统一 Toast 框架。

### FC-03 点击 Toast 跳转文件位置

未来可考虑让 Toast 支持点击交互，点击后在 Finder 中定位已保存的文件。

### FC-04 Toast 堆叠模式

未来若需要同时显示多个保存成功提示，可考虑支持堆叠模式（最多 3 条，逐条消失）。

### FC-05 自定义显示时长

未来可考虑让用户自定义 Toast 显示时长（如 1 秒 / 2 秒 / 3 秒 / 不自动消失）。
