- **问题**：D1/D3/D5/D6/R-01 含 NSPanel/NSWindow/NSAnimationContext/NotificationCenter.default.addObserver 等代码符号、属性赋值（canBecomeKey = false 等）、方法签名
- **修复**：清理所有代码符号，改用业务术语（"屏幕级透明窗口"、"应用内通知中心订阅"、"原生窗口透明度与位置动画能力"、"主线程边界"等）

### 1.3 对接契约代码符号（复审 A 发现）

- **问题**：原 1.3 节用 AutoSaveService.savedNotification / ClipMindAutoSaveSaved / eventId/fileName/skipped 等代码符号描述
- **修复**：§2.4 对接契约表改用业务术语描述（"事件标识字段"、"文件名字段"、"跳过标记字段"），并注明"具体的通知名常量、字符串值、载荷键名属于实现细节，由实现规划层核对 F2.1 实现层后落地"

### 1.4 状态名中英文混用（复审 A 发现）

- **问题**：原 4.1 节用中文状态名，原 5.3 节日志字段示例值用英文（shown/replacing）
- **修复**：状态名统一中文，日志字段示例值不再用英文枚举

### 1.5 F2.1 总开关查询接口未命名且不可 mock（复审 C 发现）

- **问题**：原 D4 称"调用 F2.1 自动保存配置模块的读取接口"但未点名，AutoSaveSettingsStore 是 final class 无 protocol，受 C-04 约束不能改 F2.1 公共接口，测试无法 mock
- **修复**：§7.4 D4 明确为"依赖注入的查询闭包（输入无、输出布尔）"，由装配方在应用入口注入具体查询实现；说明查询仅覆盖"保存完成通知派发到 Toast 显示间用户关闭总开关"的极窄时间窗；AC-06 主验证已由跳过标记过滤覆盖

### 1.6 2 秒计时与 0.2 秒动画的确定性测试机制未定义（复审 C 发现）

- **问题**：F2.1 有 D17（禁 sleep）作为可测试性决策，F2.1.1 无对应决策
- **修复**：§7.7 新增 D7 可测试性决策，明确三项：①计时器源注入协议抽象，测试用可手动推进的虚拟计时器；②动画时长作为可注入常量；③状态可观测——暴露当前状态只读属性或状态变更回调

### 1.7 失败路径描述与代码不符（复审 C 发现）

- **问题**：原设计多处称"失败路径不发送 savedNotification"，但代码实际写入失败时也发送 savedNotification(skipped:true)（并发 errorNotification）
- **修复**：§2.4 失败路径行为修正为"失败路径也发送保存完成通知（跳过标记为真），并发送错误通知触发错误弹窗"；§4.3 新增失败场景数据流（与跳过场景并列）；§4.4 失败场景不显示约束修正为"失败路径也发保存完成通知（跳过标记为真），协调模块跳过校验自然过滤"

### 1.8 第一轮非阻塞建议处理（复审 B 发现）

- D5 越界（点名 NSAnimationContext.runAnimationGroup）：已简化为"原生窗口透明度与位置动画能力"，具体 API 选型留给实现规划层
- R-01 待确认事项优先验证：§10.1 R-01 标注"优先在实现计划阶段验证"
- 双状态机同步关系：§5.4 末段新增同步关系说明 + §10.6 R-06 新增风险
- 错误处理合并动画异常分支：§8.4"动画异常"行明确合并进入/退出动画异常为统一分支
- "F2.1 F-09"引用编号错误：全文已无 F-09 引用，改用 AGENTS.md 第 10 节、NFR-006、C-06 等正确编号
- R-05 措辞偏差（.statusBar 高于 .floating）：§10.5 措辞改为"将窗口级别设置为'浮层'或更高（如'状态栏级别'）"
