## 1. Background（背景）

ClipMind 在初赛 MVP（F1.x）阶段已实现剪贴板监听、自动分类、加密存储、隐私保护与菜单栏 UI，用户复制的所有非敏感内容会自动入库到 ClipMind 历史，可被搜索与一键处理。

复赛阶段引入的新场景：用户从 ChatGPT 网页、Trae IDE、VSCode、Xcode 等 App 复制长内容（如 AI 生成的代码、报错日志、文档段落）后，往往需要把这些内容作为参考素材"喂给"另一个 AI 编程工具（典型场景：在 Trae IDE 中通过 `@文件路径` 引用文件让 Trae 读取上下文）。当前的流程是：

1. 用户在源 App 中复制长内容
2. 切换到 Finder，手动新建文件
3. 粘贴内容到文件
4. 思考文件名并保存
5. 复制文件路径
6. 切换到 Trae，在输入框输入 `@` 触发引用
7. 粘贴路径完成引用

整个过程重复且消耗专注时间，特别是当 AI 对话中需要反复引用多段长内容时，"复制 → 新建文件 → 命名 → 粘贴 → 复制路径"的循环会频繁打断思考。

本特性（F2.1）的目标是：在用户复制长内容时自动完成"保存到文件 + 路径写入剪贴板"两步，使用户切换到 Trae 后可以直接 `@` 引用，无需手动新建文件。

---

## 2. Problem Statement（问题定义）

| 编号 | 现状问题 | 可验证现象 |
|------|---------|-----------|
| P-01 | 从浏览器或 IDE 复制长内容后，用户需要手动新建文件再粘贴，重复劳动消耗专注时间 | 用户每次复制长内容平均需 5-7 步操作才能完成"保存为文件 + 复制路径" |
| P-02 | 文件命名需要用户即时思考，打断当前任务流 | 用户在保存 ChatGPT 回复时平均停留 3-5 秒思考文件名 |
| P-03 | 复制的长内容可能包含敏感信息（密码、Token），若直接保存到明文文件存在泄露风险 | 当前没有针对"保存到文件"这一动作的敏感过滤 |
| P-04 | 用户不希望所有复制都被保存为文件（如复制单词、链接、短文本），需要按场景过滤 | 现有 F1.x 流程对所有非敏感内容一视同仁入库 |
| P-05 | 不同 AI 工具对路径格式要求不同（纯路径 / file:// URI / Markdown 链接），用户需要手动改写 | Trae、Cursor、VSCode 不同插件对路径格式接受度不一致 |

## 3. Goals（目标）

### 3.1 用户可见目标

- 在白名单 App 中复制长内容时，3 秒内自动保存为文件到指定目录
- 保存完成后，剪贴板自动替换为文件路径，用户可直接在 Trae 中 `@` 引用
- 原始复制内容仍进入 ClipMind 历史，不丢失原有 F1.x 功能

### 3.2 配置能力目标

- 用户可在配置面板中启用/禁用整个自动保存功能（总开关）
- 用户可配置保存目录、白名单 App、文件格式、长度阈值、文件名长度、路径格式
- 用户可关闭敏感过滤（默认开启，与 F1.6 隐私理念一致）

### 3.3 兼容性目标

- 不影响 F1.x 既有功能（剪贴板监听、分类、加密、隐私、UI）
- 可独立启用/禁用，禁用后行为与 F1.x 完全一致
- 老版本升级时首启预置默认配置，无需用户手动迁移

## 4. Scope（范围）

### 4.1 本次包含

- 白名单 App 触发（仅白名单内 App 的复制行为才触发自动保存）
- 内容长度阈值过滤（默认 50 字，可配置）
- 文件保存到单一可配置目录（默认用户文档目录下的 ClipMind 子目录）
- 文件名前缀生成（取内容前 N 字 + 过滤特殊字符，N 可配置，默认 20 字）
- 文件名冲突自动追加序号
- 剪贴板替换为文件路径
- 原内容仍入库 ClipMind 历史
- 配置面板新增"自动保存"独立分区
- 总开关（启用/禁用整个功能）
- 敏感内容过滤（默认开，可关闭，复用 F1.6 敏感识别能力）
- 路径格式可配置（纯路径 / file:// URI / Markdown 链接）

### 4.2 本次涉及但不属于本特性单独实现

- 敏感内容识别规则（复用 F1.6 既有敏感识别能力）
- 应用 Bundle ID 识别（复用 F1.1 既有应用识别能力）
- 剪贴板写入能力（复用 macOS 系统剪贴板 API）

## 5. Functional Requirements（功能需求）

### FR-001：总开关控制

系统必须提供自动保存功能的总开关。开关开启时，符合触发条件的复制行为会被自动保存；开关关闭时，所有复制行为都不触发自动保存，行为与 F1.x 完全一致。

### FR-002：白名单 App 触发

系统必须维护一个白名单 App 集合（按应用 Bundle ID 标识）。仅当复制行为的来源 App 在白名单内时，才进入后续长度阈值检查；来源 App 不在白名单时，不触发任何自动保存行为，原内容按 F1.x 既有流程处理。

### FR-003：内容长度阈值过滤

系统必须对复制内容进行长度检查。内容字符数小于配置的长度阈值时，不触发自动保存，原内容按 F1.x 既有流程处理；内容字符数大于等于阈值时，进入后续敏感检查。长度阈值默认 50 字，可在配置面板中调整。

### FR-004：敏感内容过滤

系统必须在保存到文件前对内容进行敏感识别。识别为敏感的内容不保存到文件，原内容按 F1.x 既有流程处理（不入库）。敏感过滤默认开启，可在配置面板中关闭。敏感识别规则复用 F1.6 既有能力（密码模式、Token 格式、验证码、银行卡号、身份证号、敏感关键词）。

### FR-005：文件保存到指定目录

系统必须将符合触发条件的内容保存为文件到用户配置的目录。文件保存目录默认为用户文档目录下的 ClipMind 子目录，可在配置面板中修改。文件格式默认 Markdown，可在配置面板中切换为纯文本。

### FR-006：文件名生成

系统必须根据内容生成文件名。文件名取内容前 N 字作为前缀（N 默认 20 字，可在配置面板中调整），过滤换行符、路径分隔符与文件系统特殊字符，保留中文。文件名长度超过 N 字时截断，不足 N 字时使用全部内容。

### FR-007：文件名冲突处理

系统必须处理文件名冲突。当目标目录已存在同名文件时，新文件名自动追加序号（如 `content-1.md`、`content-2.md`），序号从 1 开始递增直到无冲突。

### FR-008：剪贴板替换为文件路径

文件保存成功后，系统必须将系统剪贴板内容替换为已保存文件的路径。路径格式可在配置面板中切换：纯路径字符串、file:// URI、Markdown 链接。剪贴板替换完成后，用户可在其他 App 中直接粘贴路径。

### FR-009：原内容仍入库 ClipMind 历史

系统必须在自动保存流程中保留原始复制内容到 ClipMind 历史。无论自动保存是否触发、是否成功，原始内容都按 F1.x 既有流程入库到 ClipMind 历史（除非命中 F1.x 的敏感过滤或黑名单）。

### FR-010：配置面板独立分区

系统必须在配置面板中新增"自动保存"独立分区。分区包含以下配置项：总开关、保存目录、白名单 App 管理、文件格式、长度阈值、文件名长度、敏感过滤开关、路径格式。所有配置项修改后立即生效，无需重启 App。

### FR-011：保存目录异常处理

系统必须处理保存目录异常情况。当保存目录不存在、无写权限或磁盘空间不足时，系统不崩溃、不替换剪贴板内容、不丢失原内容（原内容仍按 F1.x 既有流程入库）。系统通过用户可见的提示告知用户检查配置。

### FR-012：白名单 App 管理

系统必须允许用户在配置面板中添加和删除白名单 App。默认白名单包含主流浏览器与 IDE（Safari、Google Chrome、Trae IDE、VSCode、Xcode）。用户可添加任意已安装 App 到白名单，也可移除默认 App。

### FR-013：路径格式切换

系统必须支持三种路径格式输出：纯路径字符串（如 `/Users/xxx/Documents/ClipMind/Clips/content.md`）、file:// URI（如 `file:///Users/xxx/Documents/ClipMind/Clips/content.md`）、Markdown 链接（如 `[content.md](file:///Users/xxx/Documents/ClipMind/Clips/content.md)`）。默认格式为纯路径字符串，可在配置面板中切换。

### FR-014：自动保存触发时机

系统必须在剪贴板内容变化并完成 F1.x 既有捕获流程的敏感识别与黑名单检查之后、入库之前，插入自动保存检查钩子。自动保存与 F1.x 入库流程互不阻塞：自动保存失败不影响入库，入库失败不影响自动保存已完成的文件写入。

## 6. Non-functional Requirements（非功能需求）

### NFR-001：保存延迟

从用户复制到文件出现在保存目录的端到端延迟不超过 3 秒（在 M1 芯片、1000 字内容、保存目录可写的条件下）。

### NFR-002：剪贴板替换延迟

文件保存成功后到剪贴板内容变为文件路径的延迟不超过 500 毫秒。

### NFR-003：配置生效延迟

配置面板修改任意配置项后，新配置在 1 秒内对下一次复制行为生效。

### NFR-004：稳定性

保存目录异常（不存在、无写权限、磁盘满）时，App 不崩溃，不卡死，不丢失已捕获的剪贴板内容。错误恢复后无需重启 App 即可继续工作。

### NFR-005：隐私安全

敏感内容默认不保存到明文文件。保存到文件的明文内容仅存放在用户配置的目录，不上传云端，不通过网络传输。文件权限遵循 macOS 默认用户隔离（仅当前用户可读写）。

### NFR-006：性能影响

自动保存功能启用时，对 F1.x 既有剪贴板捕获流程的额外延迟不超过 100 毫秒（在 1000 字内容、保存目录可写的条件下）。

### NFR-007：日志可观测性

自动保存的关键节点（触发检查、白名单匹配、长度检查、敏感检查、文件写入、剪贴板替换、错误）必须通过项目日志系统输出结构化日志，日志不包含剪贴板原文与文件路径中的用户名。日志分类使用项目既有的日志分类体系（如新增分类须遵循既有命名规范）。

### NFR-008：可测试性

所有 FR 必须可通过自动化测试验证。涉及 UI 的 FR 必须有 XCUITest 覆盖；涉及业务逻辑的 FR 必须有 XCTest 覆盖。自动化测试不依赖真实文件系统状态，使用临时目录或 mock。

### NFR-009：兼容性

最低支持 macOS 12.4（与 F1.x 一致）。不引入新的外部依赖。与 F1.x 既有数据模型、配置模型、UI 模块兼容，不破坏既有公共接口。

### NFR-010：并发安全

自动保存流程与 F1.x 既有剪贴板捕获流程可能并发执行（用户连续复制），系统必须保证文件写入、剪贴板替换、配置读取的线程安全，不出现竞态条件导致文件损坏或剪贴板内容错乱。

## 7. Constraints（约束）

### 7.1 业务约束

- C-01：保存目录必须由用户配置，不强制固定路径。默认路径为用户文档目录下的 ClipMind 子目录，与 F1.x 加密数据库目录区分（加密数据库在应用支持目录，自动保存文件在文档目录）。
- C-02：文件格式仅支持 Markdown 与纯文本两种，不支持富文本、PDF、HTML 等其他格式。
- C-03：路径格式仅支持纯路径字符串、file:// URI、Markdown 链接三种，不支持自定义模板。
- C-04：文件名前缀长度限制在 1-50 字之间，超出范围时使用默认值 20 字。
- C-05：长度阈值限制在 1-10000 字之间，超出范围时使用默认值 50 字。
- C-06：白名单 App 以 Bundle ID 为唯一标识，同一 Bundle ID 不可重复添加。
- C-07：敏感过滤默认开启，关闭后所有非 F1.x 黑名单与敏感识别命中的内容都会被保存到文件（包括密码模式、Token 等）。关闭敏感过滤时配置面板必须显示二次确认提示。
- C-08：自动保存的文件不进入 F1.x 加密数据库，以明文形式存放在用户配置目录。用户需自行承担明文文件的管理责任（这一点必须在配置面板中提示）。

### 7.2 兼容性约束

- C-09：不修改 F1.x 既有公共接口（剪贴板条目数据模型、加密存储接口、应用配置模型公共字段、捕获流程公共回调）。
- C-10：不引入新的外部依赖（项目既有数据库依赖之外不新增第三方库）。
- C-11：与 F1.x 既有隐私理念一致：敏感内容默认不入库、不入文件；用户主动关闭敏感过滤时才允许保存敏感内容到文件。
- C-12：与 F1.x 既有 macOS 版本兼容性一致：最低 macOS 12.4，不使用 macOS 13+ 独占 API（如 NavigationStack、@Observable 宏、SwiftData）。

## 8. Acceptance Criteria（验收标准）

### AC-01：白名单 App 复制长内容触发自动保存

**Given** 用户已启用自动保存总开关，保存目录可写，白名单包含 Safari，长度阈值为 50 字
**When** 用户在 Safari 中复制一段 100 字的内容
**Then** 3 秒内保存目录出现一个 Markdown 文件，文件名取内容前 20 字（过滤特殊字符），文件内容为复制的原文；剪贴板内容被替换为该文件的纯路径字符串；原始 100 字内容仍出现在 ClipMind 历史中

### AC-02：非白名单 App 复制不触发自动保存

**Given** 用户已启用自动保存总开关，白名单不包含"备忘录"App
**When** 用户在"备忘录"中复制一段 100 字的内容
**Then** 保存目录不出现新文件；剪贴板内容保持为复制的原文（不被替换为路径）；原始 100 字内容仍出现在 ClipMind 历史中

### AC-03：白名单 App 复制短内容不触发自动保存

**Given** 用户已启用自动保存总开关，白名单包含 Safari，长度阈值为 50 字
**When** 用户在 Safari 中复制一段 20 字的内容
**Then** 保存目录不出现新文件；剪贴板内容保持为复制的原文；原始 20 字内容仍出现在 ClipMind 历史中

### AC-04：文件名冲突自动追加序号

**Given** 保存目录已存在文件 `content.md`，用户已启用自动保存总开关
**When** 用户在白名单 App 中复制一段以"content"开头且长度超过 50 字的内容
**Then** 新文件命名为 `content-1.md`；若再次复制相同内容，新文件命名为 `content-2.md`；序号递增直到无冲突

### AC-05：原内容仍进入 ClipMind 历史

**Given** 用户已启用自动保存总开关
**When** 用户在白名单 App 中复制一段 100 字的非敏感内容
**Then** ClipMind 历史列表顶部出现该 100 字内容条目，包含来源 App、时间戳、分类标签（与 F1.x 既有捕获流程一致）；该条目可被搜索、可被一键处理

### AC-06：敏感内容默认不保存到文件

**Given** 用户已启用自动保存总开关，敏感过滤开关为默认开启状态
**When** 用户在白名单 App 中复制一段包含 Token 格式（如 `sk-proj-abcdef...`）的内容，长度超过 50 字
**Then** 保存目录不出现新文件；剪贴板内容保持为复制的原文；ClipMind 历史不出现该内容条目（与 F1.x 既有敏感过滤行为一致）

### AC-07：配置面板可修改全部配置项

**Given** 用户已打开配置面板的"自动保存"分区
**When** 用户修改保存目录、添加/删除白名单 App、切换文件格式（Markdown ↔ 纯文本）、调整长度阈值、调整文件名长度、切换路径格式、关闭敏感过滤（含二次确认）
**Then** 所有修改在 1 秒内生效；下一次复制行为按新配置触发；修改后的配置在 App 重启后仍保留

### AC-08：禁用总开关后不触发任何保存行为

**Given** 用户在配置面板中关闭自动保存总开关
**When** 用户在白名单 App 中复制一段 100 字的内容
**Then** 保存目录不出现新文件；剪贴板内容保持为复制的原文；ClipMind 行为与 F1.x 完全一致（仅入库原始内容）

### AC-09：保存目录异常时弹窗提示不崩溃

**Given** 用户已启用自动保存总开关，保存目录配置为 `/nonexistent/path/`（不存在的路径）
**When** 用户在白名单 App 中复制一段 100 字的内容
**Then** App 不崩溃、不卡死；剪贴板内容保持为复制的原文（不被替换为路径）；用户可见的错误提示出现（弹窗或通知）；原始 100 字内容仍出现在 ClipMind 历史中

### AC-10：文件名过滤特殊字符保留中文

**Given** 用户已启用自动保存总开关，文件名长度为 20 字
**When** 用户在白名单 App 中复制一段以"你好世界\n这是测试/content?"开头且长度超过 50 字的内容
**Then** 生成的文件名包含"你好世界这是测试content"（换行符、路径分隔符、问号被过滤），中文保留；文件扩展名根据配置为 `.md` 或 `.txt`

### AC-11：路径格式可在三种格式间切换

**Given** 用户已启用自动保存总开关，路径格式配置为"file:// URI"
**When** 用户在白名单 App 中复制一段 100 字的内容，文件保存成功
**Then** 剪贴板内容为 `file:///绝对路径/文件名.md` 格式；切换为"Markdown 链接"后再次复制，剪贴板内容为 `[文件名.md](file:///绝对路径/文件名.md)` 格式；切换为"纯路径字符串"后再次复制，剪贴板内容为 `/绝对路径/文件名.md` 格式

### AC-12：自动保存失败不影响 F1.x 入库

**Given** 用户已启用自动保存总开关，保存目录配置为只读路径
**When** 用户在白名单 App 中复制一段 100 字的非敏感内容
**Then** 自动保存失败（文件写入错误）；ClipMind 历史仍出现该 100 字内容条目（F1.x 入库流程不受影响）；剪贴板内容保持为复制的原文

### AC-13：F1.x 入库失败不影响已完成的文件写入

**Given** 用户已启用自动保存总开关，保存目录可写，F1.x 加密数据库损坏（模拟场景）
**When** 用户在白名单 App 中复制一段 100 字的非敏感内容
**Then** 自动保存成功（文件出现在保存目录）；剪贴板内容被替换为文件路径；ClipMind 历史不出现该内容条目（F1.x 入库失败）；App 不崩溃

### AC-14：关闭敏感过滤后敏感内容可保存到文件

**Given** 用户已启用自动保存总开关，用户在配置面板中关闭敏感过滤（通过二次确认）
**When** 用户在白名单 App 中复制一段包含 Token 格式的内容，长度超过 50 字
**Then** 保存目录出现新文件，文件内容包含 Token；剪贴板内容被替换为文件路径；ClipMind 历史不出现该内容条目（F1.x 既有敏感识别仍生效，不入库）

### AC-15：白名单 App 管理可添加与删除

**Given** 用户已打开配置面板的"自动保存"分区，白名单当前包含 5 个默认 App
**When** 用户添加"备忘录"App 到白名单，并从白名单中移除"Xcode"
**Then** 白名单列表显示新增的"备忘录"与剩余 4 个默认 App；在"备忘录"中复制长内容触发自动保存；在 Xcode 中复制长内容不触发自动保存

### AC-16：配置修改持久化

**Given** 用户已修改自动保存配置（保存目录、白名单、文件格式、长度阈值、文件名长度、路径格式、敏感过滤开关、总开关）
**When** 用户退出 App 并重新启动
**Then** 所有修改后的配置保留；自动保存功能按修改后的配置工作

## 9. Out of Scope（明确不做）

以下事项明确不在 F2.1 范围内，所有"不做"条目集中于此（与第 4 节 Scope 不重复）：

- O-01：文件内容编辑（不支持在 ClipMind 中编辑已保存的文件内容）
- O-02：文件上传云端（自动保存的文件不上传到任何云端服务）
- O-03：跨设备文件同步（不支持通过 iCloud 或其他机制同步保存目录）
- O-04：文件内容 OCR（不对图片或截图内容进行 OCR 后保存为文本文件）
- O-05：富文本格式保留（不保留复制的富文本格式，统一保存为纯文本或 Markdown）
- O-06：自定义路径模板（不支持按日期、来源 App、内容类型等动态生成保存路径，仅支持单一目录）
- O-07：按 App 分目录（不支持不同白名单 App 的内容保存到不同子目录）
- O-08：文件自动清理（不实现自动清理保存目录中的旧文件，由用户自行管理）
- O-09：文件索引与搜索（不实现保存文件的全文搜索，文件搜索依赖 macOS Spotlight）
- O-10：保存历史记录 UI（不在 ClipMind UI 中展示"已保存文件列表"，保存记录仅通过日志可查）
- O-11：文件名自定义模板（不支持按日期、来源 App、内容类型等动态生成文件名，仅支持内容前缀 + 序号）
- O-12：多目录保存（不支持同时配置多个保存目录，仅支持单一目录）

## 10. Terminology（术语）

| 术语 | 定义 |
|------|------|
| 自动保存 | F2.1 引入的功能：在白名单 App 中复制长内容时，自动保存为文件并替换剪贴板为文件路径 |
| 白名单 App | 用户配置的允许触发自动保存的 App 集合，以应用 Bundle ID 标识 |
| 长度阈值 | 触发自动保存的最小内容字符数，默认 50 字，可在配置面板中调整 |
| 文件名前缀 | 根据复制内容前 N 字生成的文件名主体部分，过滤特殊字符后保留中文 |
| 文件名冲突 | 保存目录已存在同名文件时的情况，通过追加序号解决（如 `content-1.md`） |
| 路径格式 | 剪贴板替换为文件路径时的格式，支持纯路径字符串、file:// URI、Markdown 链接三种 |
| 总开关 | 控制整个自动保存功能启用/禁用的开关，关闭后行为与 F1.x 完全一致 |
| 敏感过滤 | 复用 F1.6 既有敏感识别能力，在保存到文件前对内容进行敏感检查；默认开启，可关闭 |
| 保存目录 | 用户配置的自动保存文件存放目录，默认用户文档目录下的 ClipMind 子目录 |
| 文件格式 | 自动保存文件的扩展名格式，支持 Markdown（`.md`）与纯文本（`.txt`）两种 |
| 应用 Bundle ID | macOS 系统为每个 App 分配的唯一标识符（如 `com.apple.Safari`），用于白名单匹配 |
| 原内容入库 | F2.1 自动保存流程不影响 F1.x 既有捕获入库流程，原始复制内容仍进入 ClipMind 历史 |
| 自动保存钩子 | F2.1 在 F1.x 既有剪贴板捕获流程中插入的检查点，位于敏感识别与黑名单检查之后、入库之前 |

## 11. Decision Freedom（实现自由度）

### 11.1 允许 AI 自由发挥

- A-01：自动保存服务的内部模块拆分（是否拆分为独立的白名单检查器、长度检查器、敏感检查器、文件写入器、路径格式化器，或合并为单一服务）由 AI 根据可测试性与单一职责原则决定
- A-02：文件名生成的具体过滤规则实现（使用正则表达式、字符白名单、字符黑名单等）由 AI 决定，但必须满足 AC-10 的过滤要求
- A-03：文件名冲突检测的实现方式（遍历目录、维护内存缓存、使用文件系统 API 等）由 AI 决定
- A-04：配置项的持久化方式（扩展现有应用配置模型、新增独立配置模型、使用系统偏好存储、使用加密存储）由 AI 根据与 F1.x 既有配置模型的一致性决定
- A-05：自动保存钩子在 F1.x 捕获流程中的具体插入位置（在剪贴板监听回调中、在捕获服务处理中、在独立监听器中）由 AI 决定，但必须满足 FR-014 的时机要求
- A-06：日志分类的命名（新增独立日志分类、复用既有日志分类）由 AI 根据项目既有规范决定
- A-07：配置面板 UI 的具体布局（分区位置、控件类型、交互方式）由 AI 根据既有设置面板风格决定
- A-08：错误提示的展示方式（弹窗、通知、内联错误）由 AI 根据错误严重程度与既有 UI 模式决定
- A-09：并发控制的实现方式（串行队列、Actor、异步任务）由 AI 根据项目既有并发模式决定
- A-10：单元测试与 UI 测试的组织结构（按模块分目录、按 AC 分文件、按场景分函数）由 AI 决定

### 11.2 禁止 AI 修改

- F-01：不修改 F1.x 既有公共接口（剪贴板条目数据模型公共字段、加密存储接口、应用配置模型公共字段、捕获流程公共回调、既有日志分类）
- F-02：不修改 F1.6 既有敏感识别规则（密码模式、Token 格式、验证码、银行卡号、身份证号、敏感关键词的正则规则与匹配逻辑）
- F-03：不修改 F1.1 既有应用 Bundle ID 识别逻辑（应用识别服务的实现与接口）
- F-04：不引入新的外部依赖（项目既有数据库依赖之外不新增第三方库）
- F-05：不修改 F1.x 既有数据模型（剪贴板条目、剪贴板内容、内容类型、待办项、黑名单条目的字段定义与编解码格式）
- F-06：不修改 F1.x 既有加密数据库 Schema（不新增表、不修改字段、不改变加密算法）
- F-07：不修改 F1.x 既有 UI 模块的公共接口（剪贴板状态存储、历史列表视图、详情面板、设置面板的公共接口）
- F-08：不破坏 F1.x 既有测试（不修改 ClipMindTests 与 ClipMindUITests 中既有测试用例的断言）
- F-09：不使用 macOS 13+ 独占 API（NavigationStack、@Observable 宏、SwiftData）以保持 macOS 12.4 兼容性
- F-10：不修改 F1.x 既有配置面板分区（API Key 配置、隐私设置、通用设置）的公共接口，仅新增"自动保存"分区

## 12. Future Considerations（未来扩展）

以下事项不在 F2.1 范围内，但设计时应避免堵死扩展路径：

- FC-01：按来源 App 分目录保存（未来可能支持不同白名单 App 的内容保存到不同子目录，如 `Clips/Safari/`、`Clips/Xcode/`）。设计时保存目录配置应可扩展为"目录 + 子目录规则"模型。
- FC-02：文件名自定义模板（未来可能支持按日期、来源 App、内容类型动态生成文件名，如 `2026-07-21_Safari_code.md`）。设计时文件名生成应可扩展为"模板 + 变量替换"模型。
- FC-03：保存历史记录 UI（未来可能在 ClipMind UI 中展示"已保存文件列表"，支持查看、重新复制路径、删除文件）。设计时保存记录应包含足够元数据（文件路径、原内容摘要、时间戳、来源 App）。
- FC-04：文件自动清理（未来可能支持按文件年龄、文件数量、磁盘占用自动清理保存目录）。设计时保存目录应可查询文件列表与元数据。
- FC-05：富文本格式保留（未来可能支持保留复制的富文本格式，保存为 RTF 或 HTML）。设计时文件格式应可扩展为多种格式枚举。
- FC-06：跨设备文件同步（未来可能通过 iCloud 或其他机制同步保存目录）。设计时保存目录路径应可配置为 iCloud Drive 路径。
- FC-07：文件全文搜索（未来可能在 ClipMind 中搜索保存文件的内容）。设计时保存记录应包含原内容摘要或向量，便于未来扩展搜索。
- FC-08：多目录保存（未来可能支持同时配置多个保存目录，按规则路由）。设计时保存目录配置应可扩展为"目录列表 + 路由规则"模型。
- FC-09：路径格式自定义模板（未来可能支持自定义路径格式模板，如 `<filename> at <directory>`）。设计时路径格式应可扩展为"模板 + 变量替换"模型。
- FC-10：白名单 App 推荐（未来可能根据用户复制行为推荐添加白名单 App）。设计时白名单配置应可查询与统计来源 App 复制频率。
